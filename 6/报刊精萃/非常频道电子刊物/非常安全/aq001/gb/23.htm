□...Microsoft 安全公告(MS01-016) (MS,缺陷)

涉及程序：

IIS

描述：

通过畸形构造的 WebDAV 请求导致 IIS 拒绝服务漏洞

详细：

WebDAV 是对 HTTP 的扩展，它允许远程管理 WEB 站点。在 WINDOWS2000 的协议的执行中，IIS 5.0 处理所有 WebDAV 请求的初始进程，再将合适的命令传给 WebDAV 进程。但是，在 WebDAV 处理一个畸形构造的请求时会出现漏洞。如果一连串这样的请求被发往受影响服务器则它将耗尽服务器上的CPU资源。

影响系统：

Microsoft Internet Information Services 5.0

解决方案：

改变提供 WebDAV 服务的 .DLl 权限来禁止 WebDAV 服务，具体步骤如下：

1，打开命令行

2，键入以下命令关闭IIS服务: IISRESET /STOP

3，将 Httpext.dll 文件中的ACLs设为：everyone no access

　　a.切换到文件夹%SystemRoot%\System32\Inetsrv

　　b.键入命令：CACLS httpext.dll /D Everyone

4，重启IIS服务：IISRESET /START

(附：重启 WebDAV 步骤）

1，打开资源管理器

2，进入 %SystemRoot%\System32\Inetsrv 文件夹

3，右击 Httpext.dll 文件，选择“properties”－－“security”

4，选定“everyone”单击“remove”

5，选择“Allow inheritable permissions from parent to propagate to this object ”，再单击“apply”

6，单击“OK”退出。
