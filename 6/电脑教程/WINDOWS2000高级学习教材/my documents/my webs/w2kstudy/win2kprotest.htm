Chapter 1：Installing Windows 2000 Professional

If you choose to use FAT, Windows 2000 will automatically format the partition with FAT16 if the partition is less than 2GB, If the partition is over 2GB, it will be automatically partitioned as FAT32.

The RIS server must meet the following hardware requirements:

l         It Must have the minimum processor (Pentium 133MHz or higher) and memory (128MB) required for Windows 2000 Server.

l         It must have at least two disk partitions, one for the operation system and one for RIS images. The partition that will hold the RIS images should be at least 2GB and be formatted as NTFS.

l         It needs to have a network adapter installed.

The following network services must be running on the RIS server or be accessible to RIS server from another network server:

l         Dynamic Host Configuration Protocol (DHCP) Server, which is used to assign DHCP addresses to RIS clients

l         Domain Name System (DNS) Server, which is used to locate the Active Directory

l         The Active Directory, which is used to locate RIS servers and RIS clients, as well as to manage RIS configuration settings and client installation options

As part of the RIS installation, the following services are loaded on the server:

l         Boot Information Negotiation Layer (BINL), which is used to respond to client requests

l         Single Instance Store (SIS), which manages duplicate copies of images by replacing duplicate images with a link to the original files

l         Trivial File Transfer Protocol (TFTP), which is the file transfer protocol that is used to download to Client Installation Wizard from the RIS server to the RIS clients

The user account that will perform the remote installation must have the user right to log on as a batch job.

You would use disk images if you needed to install Windows 2000 Professional on several computers that had the same configuration. You could also use disk images to quickly restore a computer to a baseline configuration that was having technical difficulties.

The System Preparation Tool is important because it strips out from the master copy information such as the unique security ID (SID) and other information that must be unique for each computer. The System Preparation Tool then runs a mini-Setup Wizard that is used to configure the following information:

l         Automatically creates a unique computer SID

l         Prompts for the end-user license agreement

l         Prompts for product ID

l         Prompts for regional settings

l         Prompts for user and company name

l         Prompts for network configuration

l         Prompts for whether the computer will be in a domain or workgroup

l         Prompts for time zone selection

The System Preparation (Sysprep) Tool is included on the Windows 2000 Professional CD in the \Support\Tools folder within the Deploy.cab file.

In order to use a disk image, the source and target computers must meet the following requirements:

l         The mass-storage controllers (SCSI or IDE) must be the same type on the source and destination computers

l         The HAL (Hardware Abstraction Layer) must be the same on the source and destination computers (this means that the processor type must be the same)

l         The size of the destination computer’s hard drive must be at least as large as the source computer’s hard drive.

l         Plug-and-Play devices between the source and destination computers do not need to match, as long as the drivers for the Plug-and-Play devices are available

If you still can’t join a domain, first install the computer in a workgroup, then join a domain after installation.

If you install Windows 2000 with the typical settings, the computer is configured as a DHCP client.

 

Chapter 2：Implementing and Conducting Administration of Resources

When you set default quota limits for new users on a volume, the quotas apply only to users who have not yet created files on that volume.

Windows 2000 Professional clients automatically download the print driver from the print server each time they send a print job. If the print driver is updated on the print server, the driver is automatically updated the next time the user sends a job to the printer.

Separator pages are used at the beginning of each document to identify the user that submitted the print job. If your printer is not shared, a separator page is generally a waste of paper. If many users share the printer, the separator page can be useful for distributing finished print jobs.

Printer Permissions:

l         Print：Allows a user a group to connect to a printer and can send print jobs to the printer.

l         Manage Printers：Allows administrative control of the printer. With this permission, a user or group can pause and restart the printer, change the spooler settings, share or unshared a printer, change printer permissions, and manage printer properties.

l         Manage Documents：Allows users to manage documents by pausing, restarting, resuming, and deleting queued documents. Users cannot control the status of the printer.

A new protocol, called Internet Print Protocol (IPP), allows users to print directly to a URL. Printer and job-related information is generated in HTML format.

In order to use a printer pool, all of your print devices must be able to use the same print driver.

FAT32 partitions can be up to 32GB, NTFS partitions can be up to 2TB. In order to convert a partition, you use the CONVERT command-line utility. The syntax for the CONVERT command is: CONVERT [drive：] /fs:ntfs.

 

Chapter 3：Hardware Devices and Drivers

Dynamic volumes cannot contain partitions or logical drives, and they are not accessible through DOS. Dynamic storage supports three dynamic volume types: simple volumes, spanned volumes, and striped volumes.

Spanned volume sets are used to dynamically increase the size of a dynamic volume. You can create spanned volumes that are formatted as FAT, FAT32, or NTFS.

In order to have full permissions to use the Disk Management utility, you should be logged on with Administrative privileges.

The Security and Quota tabs appear only for NTFS volumes.

If you decide to revert back to a basic disk, you must first delete all volumes associated with the drive.

You cannot create basic storage through Windows 2000. Basic storage only exists if you upgraded the computer from a previous version of Windows NT to Windows 2000.

When you choose to create a striped volume, you are creating a new volume that combines free space on 2 to 32 drives into a single logical partition. The free space on all drives must be equal in size. Data in the striped volume is written across all drives in 64KB stripes. (Data in spanned and extended volumes is written sequentially.)

In order to extend a volume, the simple volume must be formatted as NTFS. You cannot extend a system or boot partition.

When you add a second display adapter after Windows 2000 is installed, it will automatically become the primary display adapter. In order to use the display adapter that is built into the system board for multiple-display support, it must use the PCI or AGP standard.

Troubleshooting Multiple-Display Support：

l         You don’t see an option to Extend My Windows Desktop onto The Monitor：

Confirm that your secondary adapter is supported for multiple-display support.

Confirm that Windows 2000 is able to detect the secondary display adapter.

Try selecting the secondary adapter rather than the primary adapter in the Display Properties dialog box.

l         You don’t see any output on the secondary display：

Confirm that your secondary adapter is supported for multiple-display support, especially if you are using the built-in motherboard video adapter.

Confirm that the correct video driver has been installed for the secondary display.

Restart the computer to see if the secondary video driver is initialized. Check the status of the video adapter in Device Manager.

Try switching the order of the video adapters within the computer’s slots. See if the system will recognize the device as the primary display.

l         An application is not properly displayed：

Disable the secondary display to determine if the problem is specific to multiple-display support. Run the application on the primary display.

If you are running MS-DOS applications, try running the application in full-screen mode. For Windows applications, try running the application in a maximized Window.

Windows 2000 allows you to extend your desktop across a maximum of 10 monitors.

Depending on the BIOS on your computer, the APM tab may or may not appear. The BIOS must support a standard called Advanced Configuration and Power Interface (ACPI). If your BIOS does not support this feature, you may not see this tab.

Sample Questions

Which of the following statements regarding USB devices is not true?

A.      USB supports transmission speeds up to 20Mbps.

B.      You can connect up to 127 devices through a single USB port.

C.      You can configure your USB adapter properties through Device Manager.

D.     Common examples of USB devices include modems, printers, and keyboards.

Answer: A. USB supports transmission speeds up to 12Mbps.

If your computer is capable of supporting multiple processors, you should follow the computer manufacturer’s instructions for installing the second processor. This usually involves updating the processor’s driver to one that supports multiple processors through the Upgrade Device Driver Wizard.

To configure multiple processors, you can associate each processor with specific processes that are running on the computer. This is called processor affinity.

Configuring Multiple Processors

If you have two processors installed on your computer, you can complete the following steps to configure processor affinity.

1.       Press Ctrl+Alt+Delete and click the Task Manager button.

2.       In the Task Manager dialog box, click the Processes tab.

3.       In the Processes tab, right-click the explorer.exe process and select Processor Affinity.

4.       In the Processor Affinity dialog box, check the CPU 1 check box and click the OK button.

5.       Close the Task Manager utility.

Questions：

Which of the following steps would you need to take if you upgraded your computer from one processor to two processors in Windows 2000 Professional?

A.      Update the driver to support multiple processors.

 

Chapter 4：System Performance and Reliability

Sample Questions

When you schedule a task to be run, which of the following task properties cannot be configured?

A.      The username and password of the user who will run the task

B.      Power management, so that the task will not run if the computer is battery-operated

C.      Whether the task will be run once or will be a recurring task

D.     Whether or not another task will be run if specific conditions trigger the secondary task

Answer：D. There are no options to trigger secondary conditional tasks.

To manually synchronize a file or folder, right-click the file or folder that has been configured for offline use and select Synchronize from the pop-up menu.

Troubleshooting Offline Files

If you are trying to configure offline files and folders and you don’t see the Make Available Offline option as a folder property, check the following：

l         Are you connected to a network share on a computer that uses SMB?

l         Have you configured your computer to use offline files and folders?

l         Has the folder that you want to access been shared and, if so, do you have proper permissions to access the folder?

Know the steps required to use offline files and folders

You must first make sure that the file or folder you want to use offline is shared on the network and you have the appropriate permissions. Then use the Offline Files tab of the Folder Options dialog box to configure your computer for offline file and folder use. Now you can right-click a network file or folder and select Make Available Offline. The file or folder will be copied to the local client and you can begin using it offline.

Using Alerts for Problem Notification

The Performance Logs and Alerts utility provides another tool for proactive management in the form of alerts. Through Performance Logs and Alerts, you can specify alert thresholds (when a counter reaches a specified value) and have the utility notify you when these thresholds are reached.

Counter logs record data about hardware usage and the activity of system services. You can configure logging to occur manually or on a predefined schedule.

Trace logs measure data continuously as opposed to measuring data through periodic samples. Trace logs are also used to track data that is collected by the operating system or programs. For example, you could specify that you want to trace the creation or deletion of processes or threads.

Alerts can be generated when a specific counter exceeds or falls below a specified value. You can configure alerts so that a message is sent, a program is run, or a more detailed log file is generated.

If you are using excessive paging (swapping between the page file and physical RAM), it’s a clear sign that you need to add more memory.

PhysicalDisk > Current Disk Queue Length indicates the number of outstanding disk requests that are waiting to be processed. This value should be less that 2.

If you suspect that you have a disk subsystem bottleneck, the first thing you should check is your memory subsystem. If you do not have a memory problem, you can try the following solutions to improve disk performance:

l         Use faster disks and controllers.

l         Use disk striping to take advantage of multiple I/O channels

l         Balance heavily use files on multiple I/O channels.

l         Add another disk controller for load balancing.

Normally, you monitor and optimize the network subsystem from a networt perspective（透视）rather than from a single computer.

The following suggestions can help to optimize and minimize network traffic:

l         Use only the network protocols you need.

l         If you need to use multiple network protocols, place the most commonly used protocols higher in the binding order.

l         Use network cards that take full advantage of your bus width.

l         Use faster network cards.

During the preboot sequence, your computer accesses the NTLDR file. This file is used to control the Windows 2000 boot process until control is passed to the NTOSKRNL file for the boot sequence stage. The NTLDR file is located in the root of the system partition.

Creating the Windows 2000 Boot Disk

After you create a Windows 2000 boot disk, you can use it to boot to the Windows 2000 professional operating system in the event of a Windows 2000 Professional boot failure. You create a Windows 2000 boot disk through the following process：

1.       Format a floppy disk through the Windows 2000 Professional operating system

2.       Copy the following files from the Windows 2000 Professional System partition:

NTLDR

NTDETECT.COM

NTBOOTDD.SYS (if you use SCSI controllers with the BIOS disabled)

BOOT.INI

3.       Test the boot disk by using it to boot to Windows 2000 Professional. If the BOOT.INI file for the computer has been edited, you will need to update the BOOT.INI file on your Windows 2000 boot disk.

You can use the Event Viewer utility to track information about your computer’s hardware and software, as well as to monitor security events. All of the information that is tracked is stored in three type of log files:

l         The System log tracks events that relate to the Windows 2000 operating system

l         The Security log tracks events that are related to Windows 2000 auditing

l         Application logs track events that are related to applications that are running on your computer.

The ERD is not a bootable disk and can be accessed only by using the Windows 2000 Professional Setup CD or Windows 2000 Professional Setup Boot Disks.

Through the Recovery Console, you can perform the following tasks:

l         Copy, replace, or rename operating system files and folders. You would do this if missing or corrupt files may be causing your boot failure.

l         Enable or disable the loading of services when the computer is restarted. If a particular service may be keeping the operating system from booting, you would disable the service. If a particular service were required for successful booting, you would want to make sure that is was enabled.

l         Repair the file system boot sector or the MBR. You would use this option if a virus may have damaged the system boot sector or the MBR.

l         Create and format partitions on the drives.

You can add the Recovery Console to the Windows 2000 Professional startup options so it will be available in the event of a system failure. This configuration takes about 7MB of disk space to hold the CMDCONS folder and files. To set up this configuration：Insert the Windows 2000 Professional CD into your CD-ROM drive. From a command prompt, type WINNT32/CMDCONS.

The following are the only folders you can access through the Recovery Console：

l         The root folder

l         The %systemroot% folder and the subfolders of the Windows 2000 Professional installation

l         The CMDCONS folder

l         Removable media drives such as CD-ROM drives

If you try to access any other folders besides the ones listed above, you will receive an “access denied” error message.

In the Recovery console, you can’t copy files from a local hard disk to a floppy disk. You can only copy files from a floppy disk or CD to a hard disk, or from one hard disk to another hard disk. This is for security purposes.

 

Chapter 5：The Desktop Environment

If the configuration option is a personal preference, it is most likely a part of the user profile. Configuration options that relate to the computer are not a part of the user profile. For example, the mouse driver is not a part of a user profile, but the properties of the mouse configuration-such as the speed, pointer, and mouse button settings-reflect the user’s personal preferences and are a part of a user profile.

If the server that stores the roaming profile is unavailable, the user can log on using a local profile. If the computer containing the mandatory profile is unavailable, the user will not be able to log in until it becomes available. Only roaming profiles can be used as mandatory profile. Mandatory profiles do not work for local user profiles.

Multilanguage support consists of multilingual editing and viewing of documents and Multilanguage user interfaces, which allow the Windows 2000 Professional user interface to be presented in different languages.

Localized versions of Windows 2000 Professional do not support Multilanguage user interfaces. Localized versions only support the ability to view, edit, and print documents in other languages. Language support for the GUI is provided in Multilanguage Windows 2000 Professional.

Applications that use the MSI standard will include a file with an MSI extension on the application’s distribution media. You need to create a network share that will be used to store the application and copy the MSI file to the network share. Your next step in preparing an application for distribution is to create a Group Policy Object on a Windows 2000 Server domain controller. After you’ve created the GPO, you need to filter it so that only authorized users will be able to install the application. Finally, you need to add the package to the GPO you created for it. You can configure the package so that it is published or assigned to a user or a computer. Published applications are advertised through the Add/Remove Programs utility. Assigned applications are accessed through the Programs menu.

The Keyboard tab allows you to specify whether you want to use StickyKeys, FilterKeys, or ToggleKeys. StickyKeys allows the use of the Shift, Ctrl, or Alt key in conjunction with another key, one key at a time. FilterKeys ignores brief or repeated keystrokes. ToggleKeys makes a noise whenever you press the Caps Lock, Num Lock, or Scroll Lock key.

 

Chapter 6：Network Protocols and Services

The RAS server and the RAS client must use the same connectivity option. They must also use the same protocols and use compatible security protocols.

A virtual private network (VPN) is a private network that uses links across private or public networks (such as the Internet). When data is sent over the remote link, it is encapsulated, encrypted, and requires authentication services. In order to have a VPN, you must have a Windows 2000 Server computer that has been configured as a VPN server. With a VPN connection, all you need is access to a network such as the Internet.

In order to use Internet Connection Sharing, the following conditions must be met:

l         All of the network clients must get their IP address automatically through DHCP

l         The host computer must be configured to support Internet Connection Sharing

l         The client computers must be configured to use Internet Connection Sharing

To configure Internet Connection Sharing on the host computer, take the following steps:

1.       Create an Internet connection or a VPN connection

2.       Verify that the host computer is configured as a DHCP client and that there is a DHCP server properly configured

3.       Select Start>Settings and double-click the Network and Dial-up Connections icon

4.       Right-click the connection you want to share and select Properties from the pop-up menu

5.       The Virtual Private Connections Properties dialog box appears. Click the Sharing tab and check the Enable Internet Connect Sharing for This Connection check box. You can also select the Enable On-Demand Dialing check box.. When on-demand dialing is enabled, a connection will be automatically established if the Internet connection is not active and another computer tries to access Internet resources. Click the OK button.

The NET USE command-line utility provides a quick and easy way to map a network drive. This command has the following syntax:

NET USE x: \\computername\sharename

For example:

NET USE G: \\AppServer\AppData

You can access network shares through My Network Places or by mapping a drive in Windows Explorer.

 

Chapter 7：Implementing Monitoring, and Troubleshooting Security

CIPHER is a command-line utility that can be used to encrypt files on NTFS volumes. The syntax for the CIPHER command is:

       CIPHER /[command parameter] [pfilename]

You can use the CIPHER utility to check the encryption status of or encrypt any files or folders.

Know how to encrypt and decrypt files using EFS. In the folder Properties dialog box, select Advanced, then check the Encrypt Contents to Secure Data option. To decrypt, uncheck the Encrypt Contents to Secure Data option.

When you use the Active Directory, group policies are inherited. The order of inheritance is site, domain, and then organizational unit. Using this order, the group policy object of the organizational unit that the user or computer is a member of contains the final group policy settings.

There are three folders stored under Computer Configuration: Software Settings, Windows Settings, and Administrative Templates. Software settings policies can be used by independent software vendors to apply settings for specific applications. Through the Windows settings policies, you can configure options for scripts and for security settings. Administrative templates are used to configure options that are normally found in the HKEY_CURRENT_USER and HKEY_LOCAL_MACHINE portions of the Registry. These policies are grouped in four folders: System, Network, Printers, and Windows Components.

Questions：

You want to configure a Windows 2000 computer so that all users see a Logoff option within the Start menu. Which of the following options can be used to set the configuration?

A.     Create a customized local group policy

A username must be between 1 and 20 characters.

SIDs also ensure that if you delete and recreate a user using the same username, the new user account will not have any of the properties of the old account because it is based on a new unique SID.

Logon scripts are files that run every time a user logs on to the network. They are usually batch files, but they can be any type of executable file. To run a logon script for a user, enter the script name in the Logon Script text box in the Profile tab of the user Properties dialog box.

Local Computer Policy>computer Configuration>Windows Settings>Security Settings>Account Policies, you see two folders: Password Policy and Account Lockout Policy.

The Enforce Password History option is used so that users cannot use the same password. Users must create a new password when their password expires or changes.

Auditing allows you to track the success or failure of specific system events. You audit events that pertain to user management through the audit policies. By tracking certain events, you can create a history of specific tasks, such as user creation and successful or unsuccessful logon attempts. You can also identify security violations that arise when users attempt to access system management tasks that they do not have permission to access.

By default, auditing is not enabled, and it must be manually configured. Once auditing has been configured, you can see the results of the audit through the Event Viewer utility.

Account lockout policies, a subset of account policies, are used to specify options that prevent a user from attempting multiple failed logon attempts. If the Account Lockout Threshold value is exceeded, the account can be locked.

User who log on with a valid username and password automatically become a member of the Authenticated users special group. By default, anyone who can use the computer becomes a member of the special group Everyone. Because Brad is using the computer where his User account resides, he automatically becomes a member of the special group Interactive.

Know the most common causes of domain account authentication errors. A User might not be using a valid username or password, might have a disabled or deleted account, or might be prohibited by certain user rights. Make sure that the user is not trying to log on as a local user. Also, the computer might not be attached to a domain, or the domain controller could be unavailable.
