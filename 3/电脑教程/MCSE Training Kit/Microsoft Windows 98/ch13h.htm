Identifying a Domain

In Windows NT, a domain is a grouping of computers running Windows NT that share a common directory database of user accounts and policies (see Figure 13.3).

Figure 13.3 Windows 98 connecting to a Windows NT domain controller

As in a workgroup, all of the computers in a domain are displayed together in the domain's Network Neighborhood. Administering a domain differs from administering a workgroup, however. In a workgroup each computer running Windows 98 is responsible for its own security and administration; in a domain, one computer running Windows NT Server acts as the domain controller, which sets the security and administration policies for all other computers in the domain.

Domain Controllers

The primary domain controller (PDC) of a domain is the Windows NT Server that contains the master directory database. Every domain must have a primary domain controller. In addition, the domain may have one or more backup domain controllers (BDCs), which contain replicated copies of the directory database. Other Windows NT workstations and servers can join the domain to use domain security to protect their resources. When you log on as a domain member and/or request access to a resource on a domain member, the PDC or a BDC authenticates the logon request.

Windows 98 and Domains

Computers running Windows 98 cannot join a domain, because they cannot establish the necessary computer and domain accounts needed for a domain. When a user logs on using a computer running Windows 98, a domain controller can authenticate the user so the user can gain access to resources on domain computers.
