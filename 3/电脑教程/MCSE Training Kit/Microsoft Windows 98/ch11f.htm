Choosing a Multiple Installation Method

Three common methods for performing a multiple installation of Windows 98 on a network include CD-ROM installation, push installation, and pull installation. The method you choose will depend on your network environment.

Installing Windows 98 from CD-ROM

If you are installing Windows 98 on fewer than five computers, or if a computer's configuration varies considerably from the standard Setup in a department, then you should install from CD-ROM.

Installing from a CD-ROM is quick and convenient for one computer, and often the computer user can perform the installation. However, each installation requires that you use the CD-ROM, so installations depend on the availability of the CD-ROM. If a user has difficulty with the installation, you must provide individual assistance.

Using Push Installations for Windows 98 Setup

If you are installing Windows 98 more than five times, a push installation can make the installation process easier. A push installation sends an automated setup of Windows 98 from a network server to the client workstation. The network administrator initiates the push installation, which is forced on the target computers. Windows 98 does not include a push installation tool; however Microsoft System Management Server, a BackOffice product that runs on Windows NT Server, can perform push installations of Windows 98.

Push installations reduce the interaction between the user and the installation process, enforce corporate configuration standards, and reduce ongoing support and migration costs. When you push the installation from the server you can install Windows 98 on an individual computer or group of computers without ever touching them. You also can install Windows 98 on a computer that does not have a CD-ROM drive.

A push installation can take longer than a CD-ROM installation for a single computer, since the files must be sent over the network and may encounter heavy network traffic.

Using Pull Installations for Windows 98 Setup

If you must install Windows 98 more than five times, a pull installation can also make the process easier. In a pull installation, users request setup data from the server. You can copy the Windows 98 source files to a server, and allow users to have access to the files to download them. You can also use a batch file to automate the pull installation from the server. Multiple users can perform pull installations at the same time.

Using the pull installation method, users can install the new operating system at their convenience. They can also use this method to install Windows 98 on a computer that does not have a CD-ROM drive. However, you have less control over this type of installation than you do over a push installation.

When using a batch script for a pull installation from a server, you must create a shared directory that will contain the batch file on the server and copy the Windows 98 source files to that shared directory.
