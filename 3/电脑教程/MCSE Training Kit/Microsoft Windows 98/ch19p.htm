Troubleshooting Remote Computers

Windows 98 provides two utilities to diagnose and troubleshoot remote computers, Registry Editor and System Monitor. Registry Editor allows you to view and modify system settings in the Windows Registry. System Monitor is a performance monitoring tool that allows you to track memory and CPU usage, network I/O, and local processes and threads. To use the utilities, your computer and the remote computer must meet the following conditions:

  • Both your computer and the remote computer must have user-level access control enabled.

  • Both computers must have the Remote Registry service installed.

  • The remote computer must have Remote Administration enabled.

  • You must have administrator privileges on the remote computer.

  • The remote computer must display your user name in the Administrators list box on the Remote Administration tab in the Password Properties sheet. If the remote computer has a Windows NT security provider, the Domain Admins global group is granted access by default. If the remote computer has a Novell NetWare security provider, the Supervisor account is granted access by default.
