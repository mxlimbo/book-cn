Using Registry Checker

The registry is a database repository for information about your computer. This includes: hardware configuration, user information, and software component information.

There are several occasions in which you should back up the registry, including, for example, before editing the registry using Registry Editor, or before installing an application. You should always back up the registry before troubleshooting your computer.

Registry Checker is a system maintenance utility that finds and fixes registry problems, and backs up the registry regularly. Windows 98 provides a Windows-based utility and an MS-DOS-based utility for scanning, backing up, and restoring the registry and system configuration files.

Registry Checker backs up the registry once each day when you start the computer. If a serious problem is found in the registry, Registry Checker restores the registry from a backup copy. Registry Checker maintains as many as five backups of the registry that successfully started the computer and attempts to fix the registry if it does not locate a backup.

You can change the number of registry backups that Registry Checker maintains by editing the following line in C:\Windows\Scanreg.ini: 

MaxBackupCopies={number between 1 and 99}

ScanRegW

ScanRegW is the Windows-based Registry Checker utility. This utility scans and then backs up the following system configuration files: Registry (User.dat and System.dat), Win.ini, and System.ini. Registry Checker compresses these files and stores them in a cabinet (*.cab) file in C:\Windows\sysbckup, a hidden folder, as Rb###.cab. This scan occurs automatically when you start the system. You can run ScanRegW.exe when you want to force a scan and backup of these files.

ScanReg

ScanReg is the MS-DOS-based Registry Checker utility. If ScanRegW detects a problem with the registry, it prompts you to restart the computer to fix the problem. The restart of the computer causes ScanReg to run and restore the registry from a known good backup, or to repair the registry if no backups are available.

ScanReg has several options that are available when it is run from the command line. Windows 98 automatically invokes these options in the course of normal ScanReg operation.

Use This Option To
/backup Back up the registry and system configuration files.
/restore Choose a backup to restore.
/fix Repair the registry. Use this option only when there are no viable backups from which to restore.

Caution

Do not use backups of configuration files from Windows 95 on a Windows 98 system. If you do, the computer will not start.

Running Registry Checker

Registry Checker runs automatically during startup. If you want to run Registry Checker manually, perform the following procedure:

  • Click Start, Run, and type in Scanregw.exe.

    – Or –

  • Click Start, point to Programs, Accessories, System Tools, and click System Information. From the Tools menu, click Registry Checker.

Exporting and Importing in the Registry Editor

You use the Export and Import commands in Registry Editor to export and import portions of the registry. The export capabilities of Registry Editor save a specific branch or the entire registry in text format as a *.reg file. You can restore either the entire registry or a branch of it, by importing a *.reg file that was created by exporting the registry.
