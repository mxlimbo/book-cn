Troubleshooting with Device Manager

Device Manager lists all of the devices currently installed on your computer and provides information about the status, driver, and resources for each device. You can open Device Manager by double-clicking the System icon in Control Panel and clicking the Device Manager tab (see Figure 10.7).

Figure 10.7 The Device Manager tab in the System Properties sheet

Occasionally, Windows 98 may not be able to configure one device without creating conflicts with another device. In such a case, a message usually appears to explain what is happening and what you can do about the problem. The message may include instructions to:

  • Turn off a device to make room for the new device.

  • Disable the new device.

  • Reconfigure a legacy device to make room for the new device.

Device Manager provides a single location for you to check on the status of hardware devices and their drivers, disable or remove devices, and update device drivers.

Disabled and Problematic Devices

In Device Manager, you may occasionally see devices marked as disabled (with an X) or in a problem state (with an exclamation point).

  • A disabled device is any device that you disabled through Device Manager using the Disable in the hardware profile option.

  • A device in a problem state is a device for which Windows 98 tried to load a driver but failed.

In both cases, the device may still work if a real-mode (MS-DOS) driver is loaded in Config.sys or Autoexec.bat.

Configuring Device Properties

You can view and modify device properties by double-clicking any device listed in Device Manager. The Properties dialog box for the selected device typically contain three tabs—General, Driver, and Resources. There may be more tabs depending on the type of device. The Resources tab only appears for those devices which require IRQ, I/O addresses, DMA channels, and other resources. Other devices such as CD-ROMs and disk drives have a Settings tab instead.

  • The General tab displays the name and manufacturer of the selected device. For certain devices, you can prevent Windows 98 from loading the corresponding driver by checking Disable in this hardware profile or Remove from this hardware profile.

    • Disabling a device means that the device is enabled in the hardware profile, but the driver for the device is not loaded.

    • Removing the device from a profile means that the device is removed from the hardware list in Device Manager.

    Disabling a device is a good way to troubleshoot a device conflict. When you disable Plug and Play devices, their resources are automatically available for other devices. When you disable a device that is not Plug and Play, you must also remove the device from Device Manager and remove the hardware from your computer to free the resources.

  • The Driver tab shows specific information about the device drivers, including the provider and date. Driver File Details, which you open from this tab, lists file names, locations, and version information. The Update Driver button launches the Update Device Driver Wizard, which searches for updated drivers for your hardware. Updating a device's driver can solve some hardware problems.

  • The Resources tab displays the system resources that the device is using, such as memory addresses, I/O ports, IRQ assignments, and DMA channels. The Conflicting Device List section of this tab lists any resource conflicts.

Strategies for Resolving Conflicts

If you must resolve device conflicts manually, try the following strategies:

  • Identify a free resource, and assign the device to use that resource.

  • Disable a conflicting Plug and Play-compliant device to free its resources.

  • Disable a legacy device by removing the legacy device card and not loading the device drivers.

  • Rearrange resources used by another device to free resources needed by the device with the conflict.

  • Change jumpers on your legacy hardware to match the new settings.

Caution

Changing default settings can cause conflicts that make one or more devices unavailable on the system.
