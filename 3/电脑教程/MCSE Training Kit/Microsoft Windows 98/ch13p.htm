Enabling User-Level Security

Although you can use share-level access control to protect resources on your computer running Windows 98, anyone who discovers the password to a resource can gain access to it. If you are on a Windows NT network, you can apply tighter security by using user-level access control, which requires a valid user name and user account password in order to access resources.

Choosing a Security Provider for User-Level Access Control

When applying user-level access control to a Windows NT network, you must choose a security provider from among the domains and computers in your network (see Figure 13.8). The security provider is a Windows NT domain server or a Windows NT workgroup computer that maintains the list of users and groups to which you may assign resource permissions. When a remote user attempts to gain access to a resource on your computer, your security provider validates the user's user name and password.

You may choose only one security provider for a computer running Windows 98. This single security provider authenticates all shared resources on the computer running Windows 98.

Pass-through Authentication

When a Windows 98 server is employing user-level security, the server follows a four-step process to pass an authentication request through the network to its security provider:

  1. A remote user attempts to connect to the Windows 98 server's user-level security share, passing the remote user's user name and password.

  2. The Windows 98 user-level security server asks the Windows NT PDC security provider to verify the user name and password of the remote user.

  3. The security provider checks for the remote user's user name and password in its directory database and informs the Windows 98 server if the information is valid.

  4. The Windows 98 user-level security server grants the specified access to the remote user.

Important

A security provider must be present in order for a remote user to gain access to a computer running Windows 98 that is employing user-level access control. If a security provider is unavailable, remote users cannot have access to resources on the computer running Windows 98.

If you want to enable user-level access control, perform the following procedure:

  1. Open Control Panel and double-click the Network icon.

  2. Select User-level access control and enter the name of your security provider in the Obtain list of users and groups from list.

  3. Restart your computer in order for these settings to take effect.

Note

You cannot use both share-level and user-level security on the same computer.
