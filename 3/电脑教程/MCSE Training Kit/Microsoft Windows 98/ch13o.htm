Printing to a Windows NT Print Server

From a computer running Windows 98, you can print to a Windows NT-based print server the same way you print to a Windows 98-based print server. However, you must have Windows 98 compatible printer drivers (see Figure 13.7).

Figure 13.7 Printing to a Windows NT server

Windows 98 and Windows NT do not share printer drivers. However, if the print server is running Windows NT 4.0 or later, your computer will be able to obtain its printer driver from the print server transparently. Your computer will then configure itself automatically if the Windows 98 printer drivers have been installed on the server. If the drivers have not been installed on the server, you will be prompted for a manufacturer and model; you must supply that information and the Windows 98 source files or third party driver disk.

If the Windows 98 printer driver has been updated on the print server, you must remove and reinstall the printer icon from your Printers folder to obtain the new printer driver.

Note

For more information about printing in a network environment, refer to Chapter 12, "Configuring Windows 98 for Use on a Network," in this course.

Remote Printer Administration

You can administer a remote Windows NT printer the same way you administer a remote Windows 98 printer; however, depending on the permissions assigned to the Windows NT printer, some commands may not be available.
