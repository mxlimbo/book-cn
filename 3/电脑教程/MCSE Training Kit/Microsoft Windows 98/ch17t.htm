Connecting to the Internet through a Proxy or Firewall

Your network administrator can use two tools to regulate a site's Web traffic: firewalls and proxy services.

A firewall is a mechanism that can examine every packet and can filter traffic based on administratively specified policies. Firewalls support extensive security auditing, such as:

  • Generation of alarms based on filtered traffic (an alarm is a visual or auditory signal from a computer alerting the user to a problem)

  • Detection of common security violations

  • Monitoring of IP address usage

In networking terms, a proxy server computer acts on behalf of several client computers that are requesting content from the Internet or elsewhere on an intranet. In this case, the proxy server is the secure gateway to the Internet for several client computers (see Figure 17.5).

Figure 17.5 Using Microsoft Proxy Server

The proxy server is nearly transparent to the user and the Internet resource in this communications path. The user interacting with the Internet at a desktop PC usually cannot tell that a proxy server is interceding, unless the user attempts to gain access to a service or to a site that the proxy server is disallowing. The Web server being accessed across the Internet interprets the requests from the proxy server as requests from a browser or FTP client.

If you want to configure Internet Explorer to use a proxy server, perform the following procedure:

  1. From Control Panel, double-click the Internet icon.

  2. On the Internet Properties page, select the Connection tab.

  3. Click Access the Internet using a proxy server and type the address and port.

  4. Click Bypass proxy server for local (Intranet) addresses in order to access intranet resources without going through the proxy server.

  5. If you need to add additional proxy servers for different protocols or functions, or if you wish to bypass the proxy server for particular addresses, click Advanced.

  6. Click OK to save the settings.
