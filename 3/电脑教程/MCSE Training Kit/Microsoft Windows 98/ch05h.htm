Practice: Viewing Current Tasks

In this practice, you will use the Windows 98 Task Manager to see a list of tasks running on your computer.

    To see a list of tasks running on your computer

    1. From the Windows 98 Start menu, click Run, type taskman, and click OK.

      The Windows 98 Task Manager window appears displaying each task currently running on your computer.

    2. From the Windows 98 Start menu, point to Programs, Accessories, and click WordPad.

      Wordpad is added to the list of running tasks in Task Manager.

Processes and Threads

Unlike 16-bit applications, Win32-based applications do not need to yield to other running applications to multitask properly. Instead, they can take advantage of multithreading, a mechanism that Windows 98 provides to facilitate the ability to run applications concurrently.

A Win32 application running in the system is called a process in terms of the operating system. Each process consists of at least a single thread of execution that identifies the code path flow as it is run by the operating system. A thread is a unit of code belonging to a process that is allocated a time slice from the operating system.

Win32 applications can initiate multiple threads for a given process to enhance the application for the user by improving throughput, enhancing responsiveness, and aiding background processing. Because of the preemptive multitasking feature of Windows 98, threads of execution allow code from various applications to be smoothly processed.

For example, the Windows 98 shell (Explorer.exe) is a Win32-based process. Each folder window that opens is a separate thread of execution. Therefore, when you initiate a copy operation between two shell folder windows, the operation is performed on the thread of the target window. During this operation, you can still use other windows in the Windows 98 shell without interruption. You could also start a different copy operation between two other windows.

The Process Scheduler

The Process Scheduler is the Windows 98 component responsible for providing system resources to applications and other processes. It also provides scheduling for processes to allow multiple applications to run concurrently.

The Process Scheduler gives the illusion that all threads are running concurrently by providing time slices (called quantums) to the threads. For example, if two threads are running on the system, the first thread would execute for a period of time and the second thread would execute immediately after. When the second thread's time slice expires, the first thread would begin executing again.
