Using System Policy Editor

System Policy Editor is a tool for creating and modifying system policy files with templates. You must install System Policy Editor after Windows 98 installation.

If you want to install System Policy Editor, perform the following procedure:

  1. Double-click the Add/Remove Programs icon in Control Panel.

  2. Click the Windows Setup tab of the dialog box.

  3. Click Have Disk.

  4. Insert the Windows 98 installation disk.

  5. Locate the System Policy Editor (Poledit) folder on the installation disk, and then in the Copy manufacturer's files from box, type the drive letter and path, (for example: D:\tools\reskit\netadmin\poledit where D is the name of your CD-ROM drive), and click OK.

  6. The Have Disk dialog box (see Figure 16.2) appears, listing the component for System Policy Editor installation. Select the System Policy Editor check box and click Install.

Installation of System Policy Editor adds a shortcut to the System Tools menu.

You can create a system policy that only contains settings for Default User or Default Computer, or you can add entries to system policies for specific users, groups, and computers with the Edit menu in System Policy Editor.

User Policies

A user system policy applies registry settings when a specific user logs on. When policies exist for a user and also for one or more groups to which the user belongs, only the settings in the user policy apply. If there is no user or group policy for a specific user, the Default User policy (if it exists) applies.

Group Policies

Group policies make it possible to add specific restrictions to Windows NT Directory Services (NTDS) global groups or NetWare groups. However, you must follow these guidelines:

  • You must install group policy support (Grouppol.dll) on each computer that you use to create group policies and on each computer to which those policies will apply. You select group policy support as an option in a custom Windows 98 installation, or you can install group policy support through the System Tools category of components on the Windows Setup tab using the Add/Remove Programs icon in Control Panel.

  • You must use a central security server, such as a Windows NT domain or a NetWare server, with group policies.

  • Group policies apply only to existing Windows NT global groups or Novell NetWare groups. You cannot create new groups with System Policy Editor.

Note

Computers without Grouppol.dll installed do not process group policies. However, you can still use the same Config.pol file for all computers; the computers that do not support group policies disregard that section of the policy file.

Setting Group Priorities

Group policies download in order from the lowest priority group to the highest priority group. The group with the highest priority processes last, so settings for this group overwrite settings defined in the other group policies.

If you want to set priority levels for group policies, perform the following procedure:

  1. In System Policy Editor, click File then Open Policy.

  2. Locate the Config.pol file and click Open.

  3. In the Options menu, click Group Priority.

  4. In the Group Priority dialog box, click a group, and then use the Move Up and Move Down buttons to set the desired priority. Groups highest in the list have the highest priority.

Computer Policies

Computer policies define settings for a default computer or for specifically named computers. The default computer settings apply when a user logs on to a computer that does not have individual policies assigned.

Computer settings in system policies prevent users from modifying the hardware and environment settings for the operating system, ensuring that Windows 98 starts up in a predictable way. You can set options to restrict access to computer-specific system and network features. When you apply system policies to a computer, they affect all users of that computer.
