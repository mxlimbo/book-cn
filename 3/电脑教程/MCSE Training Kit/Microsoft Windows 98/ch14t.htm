Installing and Configuring File and Printer Sharing for NetWare Networks

With Windows 98, you can use File and printer sharing for NetWare Networks to share resources with NetWare clients. To a NetWare client, even an MS-DOS client running the netx requestor, your computer appears as a NetWare server.

File and printer sharing for NetWare Networks has the following requirements:

  • IPX/SPX-compatible Protocol must be installed. If it is not installed, it installs automatically with File and printer sharing for NetWare Networks.

  • Client for NetWare Networks must be installed. If it is not installed, it installs automatically with File and printer sharing for NetWare Networks.

  • A bindery-based NetWare server must act as a security provider. File and printer sharing for NetWare Networks does not support share-level security or NetWare Directory Services.

Important

File and Printer Sharing for Microsoft Networks must not be installed. Windows 98 will not install File and printer sharing for NetWare Networks until File and Printer Sharing for Microsoft Networks is removed.

Installing File and Printer Sharing for NetWare Networks

You can install File and printer sharing for NetWare Networks from the Network option of Control Panel.

If you want to install File and printer sharing for NetWare Networks, perform the following procedure:

  1. In Control Panel, click the Network icon.

  2. On the Configuration tab, click Add.

  3. Select Client and click Add.

  4. Select Microsoft and Client for NetWare Networks.

  5. Click OK, and then click OK again.

After installing the service, you must set the access control level to user-level access. To set the access control level, follow these steps:

  1. In Control Panel, double-click the Network icon.

  2. On the Access Control tab, click User-level access control.

  3. In the Obtain list of users and groups from box, type in a security provider.

  4. Click OK.

  5. Restart your computer for the changes to take effect.

Configuring File and Printer Sharing for NetWare Networks

File and printer sharing for NetWare Networks has two configurable properties: Service Advertising Protocol (SAP) advertising and workgroup advertising.

SAP Advertising

Only real-mode NetWare clients require SAP advertising. With this setting enabled, the Windows 98 computer uses SAP to advertise its services every 60 seconds. The computer appears on the list of NetWare servers returned by the nlist server /b command. Because this setting causes additional broadcast traffic and overloads the SAP browsing model, use SAP advertising only with NetWare clients that require the property.

Because Windows 98 clients do not rely on SAP, this setting is disabled by default.

Workgroup Advertising

With this setting enabled, the Windows 98 computer advertises its service using the Microsoft browser broadcasts. The computer appears with the rest of the workgroup servers in Network Neighborhood, but does not appear on the list of servers reported by the nlist server /b command. If you do not configure your computer for SAP advertising, a client running netx or vlm cannot browse or connect to the computer. The default setting is Enabled: May be Master, which means your computer may be elected to be the computer that maintains the workgroup server list. Unless you understand the browsing process on Microsoft networks, you should not change this setting.

Note

For more information on browsing, refer to the Microsoft Windows 98 Resource Kit.
