Monitoring the Boot Process

The first time you boot Windows 98 following a successful setup, Windows 98 creates a Bootlog.txt file (see Figure 10.5) in the root of your system partition, with each step of the boot process immediately logged as it occurs.

Figure 10.5 The Bootlog.txt file

The next two times you start your computer, Windows 98 renames the Bootlog.txt file to Bootlog.prv and creates a new Bootlog.txt file. After the third startup, Windows 98 creates a new Bootlog.txt file only if you select the Logged option from the Startup Menu.

You can use Bootlog.txt to diagnose boot problems such as:

  • Driver Load Failures: The first part of Bootlog.txt records whether drivers were successfully loaded. Since no drivers are yet initialized, a failure in this section simply means the driver was not found.

  • Driver Initialization Failures: The next part of Bootlog.txt records critical system drivers. The initialization of these drivers is recorded under the heading SYSCRITINIT. Because no file I/O is allowed during initialization of these drivers, all SYSCRITINIT records are written following the complete initialization of all SYSCRITINIT drivers. The initialization (or lack thereof) of each non-SYSCRITINIT driver is recorded as it occurs.

    If Bootlog.txt does not record any SYSCRITINIT entries, then at least one VxD failed during its SYSCRITINIT phase.

If you want to determine which device failed, perform the following step:

  1. Add LogSysCritInit=1 to the [386enh] section of System.ini.

    This causes VxDs to display individually during the SYSCRITINIT phase at the top of the screen (press Esc to bypass the logo so you can see this).

    Typically, you only see a SYSCRITINITSUCCESS=SHELL, since SHELL is the last VxD. If a VxD is causing the system to stop, the display shows the name of that VxD instead of SHELL.
