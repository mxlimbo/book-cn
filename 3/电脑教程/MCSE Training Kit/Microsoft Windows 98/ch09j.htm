Using ScanDisk

ScanDisk is a full-featured disk analysis and repair program. ScanDisk in Windows 98 is upgraded to recognize and fix issues with FAT32 drives and compressed drives. ScanDisk runs as part of Windows 98 Setup. You can also run ScanDisk manually at any time after Windows 98 Setup. ScanDisk runs automatically after an improper shutdown unless it is disabled through Msdos.sys. You can do this by setting the AutoScan value to zero.

Windows 98 provides two versions of ScanDisk: a graphical Windows-based version (Scandskw.exe) and an MS-DOS-based version that is contained in the Windows\Command folder (Scandisk.exe).

ScanDisk checks and fixes problems in the following areas on hard disk drives, floppy disk drives, RAM drives, and memory cards:

  • FAT16 partitions, and FAT32 partitions up to 127.53 GB

  • Long file names

  • File system structure, such as lost clusters, cross-linked files, invalid file names and dates

  • Directory tree structure

  • Physical surface of the drive (marks bad sectors so that they will not be used. Moves data stored in a bad cluster if possible)

  • DriveSpace 3 or DoubleSpace volumes

ScanDisk cannot find or fix errors on CD-ROM drives, network drives, or drives created by using assign, subst, join, or interlnk.

If you want to run ScanDisk, click Start, point to Programs, Accessories, System Tools, and then click ScanDisk. The opening window of ScanDisk will list disk drives to scan on your computer, and the types of tests to perform on these drives.

You can run a Standard or a Thorough test with ScanDisk.

  • Standard test checks files and folders for errors.

  • Thorough test performs the Standard test and checks the surface of the hard disk for errors. The Thorough test has the following options for Areas of the disk to scan:

    • System and data areas

    • System area only

    • Data area only

    Your other Thorough test options are:

    • Do not perform write-testing. With write-testing, ScanDisk reads the contents of each sector on the hard disk, then writes the contents back to verify that the disk can be read from and written to correctly.

    • Do not repair bad sectors in hidden and system files. ScanDisk repairs bad sectors by moving the data they contain to a valid location. If you select this option, ScanDisk does not move data that is part of a hidden or system file, some of which are position-dependent.
