Generating a Setup Script

You can use or modify an existing setup script, or you can create a new setup script by performing a custom setup.

Editing a Setup Script

If you wish to change any of the values for the items in a setup script, you can edit the script using a text editor. You may also want to create alternate or more detailed setup scripts by editing existing scripts. The following are editing guidelines for the setup script file:

  • Each section starts with a unique section heading enclosed in brackets ([ ]).

  • Each section can contain one or more entries. The typical entry consists of a key name, an equal sign, and a value.

  • Key names within a section need not be unique. Key names have defined values. Set the value for optional components to be installed as "1". A value of "0" means the component will not be installed.

  • Comments can be included on a line after the value, and begin with a semicolon. Setup ignores all text after a semicolon until the next line.

Note

If you have existing setup scripts for Windows 95 or Windows 3.1x, they can be edited to include the new components available with Windows 98.

Creating a New Setup Script

Creating a new setup script allows you to direct the Setup program to configure settings to your specifications. If you want to create a new setup script, perform the following procedure:

  1. Perform Setup on a newly formatted hard disk, and then select the Custom Setup option.

  2. Select the components that you want to install.

  3. After you complete Setup, use a text editor to copy the following sections from the Setuplog.txt file, located in the root directory of the C drive: 

[Setup]
[OptionalComponents]

Running Setup with a Script

To run Setup with an installation script, specify the path to the script file on the command line. The following example shows how to run Setup using a setup script named Script.inf: 

D:\Win98\Setup.exe A:\Script.inf

–or– 

\\Server\Win98\Setup.exe \\Server\Scripts\Script.inf

Note

Setup scripts can be given any MS-DOS 8.3 filename with an .inf extension.

After you create a setup script, you can install Windows 98 automatically by running Setup at each client computer, using the name of the setup script as a command-line parameter. You can run Setup from any of the following:

  • A login script.

  • Server-based system management software.

  • A batch file, distributed on disk or by e-mail, that contains the appropriate setup command line.
