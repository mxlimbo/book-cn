Troubleshooting NetWare Interoperation

Some common troubleshooting scenarios related to NetWare interoperation that you may encountered are presented in the following table:

Problem Probable Cause/Solution
A Windows 98 user attaches to a NetWare server as a guest, then receives an account, but cannot use the new account to attach to the NetWare server. When attaching to a NetWare server as a guest, you have an option to save the guest user name and password in your password list. Whenever you reattach to that server, the server applies the username and password from your password list.

To update your passwords list with your new account information:

1. Right-click the server's icon and click Attach as.

2. Enter your new credentials and check the Save this password in your password list check box.

User cannot access resources on a Windows 98 server running File and printer sharing for NetWare Networks. The security provider is unavailable. A security provider must be present to authenticate logon requests to a Windows 98 server running File and printer sharing for NetWare Networks.
The logon script does not execute completely. Terminate and Stay Resident (TSR) programs and NetWare 4.x-specific commands are not processed by the Windows 98 logon script processor. Load the TSRs in Autoexec.bat or Winstart.bat instead.
The NetWare Supervisor cannot administer the server by running Syscon on a Windows 98-based computer. Syscon was invoked through Network Neighborhood, or by a direct UNC command. Many NetWare utilities will run onlywhen they are run from a mapped drive, or when a mapped drive is in the search path.
