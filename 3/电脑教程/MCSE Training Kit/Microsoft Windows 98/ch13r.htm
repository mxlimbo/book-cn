Assigning Folder Permissions

When you share a printer in Windows 98 using user-level access control, the ACL has to contain only the list of users and groups permitted to use the printer. When you share a folder, however, the ACL must also contain folder permissions for each entry on the ACL.

Permissions specify what a user or group can do in a folder on a Windows 98 server. Permissions apply only to folders, printers, and other shared objects, but not to files. As with share-level access control, you can assign Read-Only and Full Access rights, but unlike share-level access control, which grants access to anyone with the password, you can assign different rights to individual users.

You can also specify Custom Access Rights, which define what users can do in a shared folder (see Figure 13.10).

Figure 13.10 Access Rights options

Custom access rights can be a combination of the following:

With This Custom Access Right A User Can
Read Files Open and read files in the folder.
Write Files Write to existing files in the folder.
Create Files and Folders Create new files and subfolders in the folder.
Delete Files Delete existing files and subfolders.
Change File Attributes Rename files and subfolders and modify file and subfolder attributes.
List Files View a list of files in the folder.
Change Access Control Modify permissions (but Windows 98 does not include a utility to do this).

Caution

In most cases, grant either full or read-only access rather than assign custom.
