Summary

The following information summarizes the key points in this chapter.

Preparing for Installation

  • Before installing Windows 98 you should determine if your computer meets at least minimum requirements, such as processor, memory, and free disk space.

  • It is important to disable any software that can cause conflicts during Setup, such as anti-virus software and memory managers.

  • Windows 98 can be dual booted with Windows NT or Windows 3.1/MS-DOS, but not Windows 95.

  • You can install Windows 98 by upgrading an existing Windows 95 or MS-DOS operating system. You also can install Windows 98 on a newly formatted hard disk or to a new folder.

Upgrading from Windows 95

  • The Setup process is organized in five phases. In Phase 1—Preparing to run Windows 98 Setup, Setup checks for anti-virus software and prompts you to disable it in case any is found.

  • In Phase 2—Collecting information about your computer, Setup runs ScanDisk, checks the registry for corrupt files, and prepares your Windows directory. Setup also prompts you to save your existing MS-DOS and Windows system files, and prompts you to create an Emergency Startup Disk.

  • In Phase 3—Copying Windows 98 files to your computer, Setup copies Windows 98 files to your computer. No user input is required during the file-copying phase.

  • In Phase 4—Restarting your computer, your computer is automatically restarted. Win.ini, System.ini, and registry files are edited to add Windows 98-specific settings. Autoexec.bat and Config.sys are examined for TSRs and incompatible drivers.

  • In Phase 5—Setting up hardware and finalizing settings, Setup detects Plug and Play devices and legacy hardware. Setup finalizes settings for system configuration, Control Panel, Start menu, Windows Help, and MS-DOS programs. Finally, Setup restarts your computer.

Installing from MS-DOS

  • When you install Windows 98 from MS-DOS, Setup follows the same five phases as upgrading from Windows, but some additional steps are added to the process.

  • In Phase 1—Real-Mode ScanDisk runs and the Mini.cab file is copied.

  • In Phase 2—you must choose the Install Directory, select Setup options, and supply user and computer information.

  • In Phase 4—legacy hardware is detected.

Automating Setup

  • Setup can be automated by using a setup script. A setup script is a text file that contains predefined settings for all of the options specified during Setup, and can contain instructions for installing additional hardware.

  • You can generate a script by installing Windows 98 to a newly formatted hard disk and using a text editor to copy the [Setup] and [Optional Components] sections from the Setuplog.txt file, located in the root directory of the C drive. You also can edit scripts from previous Windows installs.

  • To run a Setup with a script, specify the path to the script file on the command line.

Troubleshooting with Windows 98 Setup

  • Failures can occur during installation of Windows 98, and Setup uses different procedures to recover from a Setup failure, depending on the point where the failure occurs in the Setup process.

  • If failure occurs before the initial restart, you must turn your computer off and on and start Setup again.

  • If failure occurs during hardware detection, you must turn your computer off and on, and then use the Device Manager to determine which of your hardware components were detected.

  • Other installation support issues involve incompatible video drivers, extraneous copies of Setupx.dll remaining in memory, adding or removing installed components, interrupting file copying, and not disabling virus protection software.

Uninstalling Windows 98

  • It is possible to uninstall Windows 98 and return your computer to its previous Windows 95 configuration if you have clicked Yes (recommended) on the Save System Files screen during Setup, not compressed your system or boot partition after installing Windows 98, and not converted any hard disks to FAT32.
