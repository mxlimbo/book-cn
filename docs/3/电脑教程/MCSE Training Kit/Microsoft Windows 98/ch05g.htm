Multitasking in Windows 98

Users often need to work with multiple applications at the same time. Windows 98 provides this functionality by multitasking applications. Windows 98 uses two types of multitasking: cooperative multitasking and preemptive multitasking.

With Windows 3.1x, applications run concurrently through a method known as cooperative multitasking. Using this method, the operating system requires an application to check the message queue periodically to relinquish control of the system to other running applications. The message queue in Windows 3.1x is a central location in the operating system where applications could receive event messages such as a mouse click or input from the keyboard. Applications that do not check the message queue will block other applications, keeping them from executing. This also prevents users from switching to other applications. For compatibility reasons, Windows 98 implements cooperative multitasking for 16-bit applications.

Windows 98 uses preemptive multitasking for Win32-based applications. This means that the operating system has complete control over running applications. To achieve this, the operating system schedules a certain amount of time for each application to execute.
