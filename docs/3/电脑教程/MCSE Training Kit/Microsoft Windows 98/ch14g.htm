Installing and Configuring the IPX/SPX-compatible Protocol

The first step in connecting to a NetWare network is installing the IPX/SPX-compatible protocol.

IPX/SPX is the primary protocol used by Novell NetWare networks. To set up interoperation with any version of NetWare, you must install and configure IPX/SPX-compatible Protocol. IPX/SPX-compatible Protocol is fully compatible with both the IPX/SPX protocol and the NWLink protocol that Windows NT uses. Generally, only computers that must interact with NetWare clients or servers use the IPX/SPX-compatible Protocol.

Installing the IPX/SPX-compatible Protocol

If you want to install the IPX/SPX-compatible Protocol through the Network option of Control Panel, perform the following procedure:

  1. In Control Panel, double-click the Network icon.

  2. On the Configuration tab, click Add.

  3. Click on Protocol and click Add.

  4. Click on Microsoft and then select IPX/SPX-compatible Protocol.

  5. Click OK, and then click OK again.

    Windows 98 copies files needed to support IPX/SPX-compatible Protocol.

  6. Restart your computer.

Configuring the IPX/SPX-compatible Protocol

You can bind IPX/SPX-compatible Protocol to only one network adapter plus the dial-up adapter used for Dial-Up Networking. Binding is the process that establishes the communication channel between a protocol driver and a network adapter. The IPX/SPX-compatible protocol has several configurable parameters in its Properties list, found on the Advanced tab.

The following table describes the parameters for different properties:

Configure This Property In Order To
Force Even Length Packets Maintain compatibility with older NetWare Ethernet drivers that discard any odd-length frames they receive. By default this option is disabled.
Frame Type Specify the frame type. By default, this option is set to auto which autodetects a frame type the next time (and only the next time) the computer restarts. If no frame type is detected, Windows 98 defaults to 802.2; if multiple frame types are detected, Windows 98 selects the most prevalent one.
Maximum Connections Specify the maximum number of simultaneous SPX connections supported. Defaults to 1 in a range from 1 to 128.
Maximum Sockets Specify the maximum number of simultaneous IPX sockets supported. Defaults to 2 in a range from 1 to 128.
Network Address Specify an IPX network ID number. Defaults to 0, which indicates that Windows 98 will autodetect a network ID number at startup.
Source Routing Enable source routing to destination workstations on a token ring network.

You can configure your computer to be a client to a NetWare server in one of two ways, depending on which version of NetWare is installed.

  • If your servers run NetWare 3.x, or if your servers run NetWare 4.x in bindery emulation mode, install Client for NetWare Networks.

  • If your servers run NetWare 4.x with NetWare Directory Services (NDS), install Client for NetWare Networks and Service for NetWare Directory Services.

If you want to configure the IPX/SPX-compatible Protocol, perform the following procedure:

  1. In Control Panel, double-click the Network icon.

  2. On the Configuration tab, select IPX/SPX-compatible Protocol.

  3. Click on Properties.

  4. Click the Advanced tab (see Figure 14.2).

  5. Select the property you want to configure.

  6. Click OK.

After installing the client and, if necessary, Service for NetWare Directory Services, you must configure the logon functions you want to use. In addition, you must consider how to set up the logon procedure for your users.
