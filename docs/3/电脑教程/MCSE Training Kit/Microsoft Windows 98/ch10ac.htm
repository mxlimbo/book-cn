Using Dr. Watson

A General Protection Fault (GPF) is a computer condition that causes a Windows application to stop working. GPFs most commonly occur when an application tries to use memory assigned to another application.

Dr. Watson collects detailed information about the state of your system when a GPF occurs (see Figure 10.8). Dr. Watson intercepts the software fault, identifies the software that faulted, and offers a detailed description of the cause.

Figure 10.8 Identifying a GPF with Dr. Watson

Dr. Watson stores the information it collects in a log file. The Details button in the dialog box that appears when a GPF occurs displays the same information on screen.

Note

You can enter information in the Dr. Watson dialog box about what you were doing when the problem occurred. Be sure to enter detailed information about the problem, such as the applications and processes that were running when the GPF occurred, to help technical support troubleshoot the problem.

This information is used by technical support engineers to troubleshoot GPFs. If you cannot determine the cause of persistent GPFs, you should generate a Dr. Watson log file and send it to technical support. By examining the log file, technical support may be able to help you determine the cause of the problem.

If you want to start Dr. Watson, perform the following step:

  1. Click Start, Run, and type drwatson.exe.

    The Dr. Watson icon appears in the status bar portion of the taskbar. This indicates that Dr. Watson is monitoring the system for GPFs.

Note

Dr. Watson must be running in order to capture information about any GPFs that occur.

When a GPF occurs, Dr. Watson generates a log file named Watsonxx.wlg (where xx is a number). By default, Dr. Watson log files are saved to the C:\Windows\Drwatson folder.

The Dr. Watson File menu includes a Save option you can use to produce a log file manually. This is helpful in situations when you can reproduce the problem. In this situation, you would start Dr. Watson, save the snapshot of your system in a log file, reproduce the problem, and then save the log that results. Send both logs to technical support to give them a before and after view of your system.

You can customize the following options in Dr. Watson on the View menu through the Options button.

  • Log Files configures the number of log entries to save, and designates the folder where they will be saved.

  • Disassembly configures the number of CPU instructions and stack frames that are reported in the log file. Only configure these options if technical support requests a change in the setting.

  • View sets the display to Standard View or Advanced View when you run Dr. Watson (see Figure 10.9).

Figure 10.9 Dr. Watson Standard View and Advanced View

Standard View

Standard View contains only the Diagnosis tab. This tab provides a description of the problem, and a box where you can enter detailed information about the state of the system when the problem occurred.

Advanced View

Advanced View contains the tabs that are shown in the table. This view is most useful to technical support personnel.

Tab Contains
Diagnosis What caused the GPF to occur and a suggested solution. Also provides a text field for entering detailed information, such as what you were doing when the GPF occurred.
System The version of Windows, install type, Internet Explorer version, uptime, start up mode (normal or safe), computer name and current user, CPU type, amount of RAM, percentage of free system resources, swap file details, and temporary files location.
Tasks Applications that were in memory when the GPF occurred.
Startup Applications that are launched at startup.
Kernel Drivers Installed Kernel mode (ring 0) drivers.
User Drivers Installed user mode (ring 3) drivers.
MS-DOS Drivers Installed MS-DOS drivers.
16-bit Modules 16-bit modules in memory when the fault occurred.
Modules* 32-bit modules the application was accessing when the fault occurred.
Detail* Technical details (CPU instructions, stack dumps, and so on) for technical support personnel.
* Only visible when a GPF has occurred.
