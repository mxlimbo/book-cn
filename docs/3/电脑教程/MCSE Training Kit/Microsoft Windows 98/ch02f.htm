Preventing Software Conflicts During Setup

Before you install Windows 98, you must disable software that may cause problems during Setup. Software that may disrupt Setup includes anti-virus software and memory managers.

Disabling Anti-Virus Software

Windows 98 Setup makes changes to the boot sector of all hard disks in the computer. Because virus protection software is designed to prevent changes to the boot sector, it often can cause Setup to fail. Therefore, be sure that automatic anti-virus protection software and BIOS-level anti-virus protection are disabled.

Disabling Memory Managers

When you install Windows 98 from MS-DOS or Windows 3.1x, you should remove these memory managers from your Config.sys file:

  • EMM386 (only if using highscan option)

  • QEMM

  • 386MAX

Note

If Setup detects conflicting software, it provides instructions for correcting the problem. If Setup presents an error message that states that Setup has failed, restart your computer and Setup will resume automatically.
