Using the Startup Menu

Windows 98 offers several startup options to assist users in controlling and troubleshooting the boot process. These options are available on the Startup menu by pressing and holding CTRL during the startup process. Some menu items also have a function key associated with them that can be pressed to bypass the Startup menu.

Press To
CTRL Display the Start Menu.
F4 Start the previous version of MS-DOS, if available.
F5 Start Windows 98 in Safe Mode.
ALT+F5 Start at Safe Mode Command Prompt.
SHIFT + F5 Start at Command Prompt.
F8 Display the Start Menu.
SHIFT + F8 Start Windows 98 with Step-by-Step Confirmation.

The following table describes the Startup menu options. The contents of this menu can vary, depending on the options specified in the Msdos.sys file, and the configuration of the computer.

Startup Menu Options

Startup Menu Option Description
Normal Starts Windows 98, loading all normal startup files and Registry values.
Logged (Bootlog.txt) Starts Windows 98 normally, updating a log file called Bootlog.txt at each step along the way. The boot process will take considerably longer because of the logging.
Safe Mode Starts Windows 98, bypassing startup files and using only basic system drivers. You can also start this option by pressing F5 or typing win /d:m at the command prompt.
Step-by-Step Confirmation Starts Windows 98, requiring you to confirm startup files line by line. You can also start this option by pressing SHIFT+F8 when the Startup menu is displayed.
Command Prompt Only Starts the Windows 98 operating system, loading Himem.sys, Ifshlp.sys, Config.sys, and autoexec.bat, but does not load protected-mode Windows 98. You also can start this option by pressing SHIFT+F5.
Safe Mode Command Prompt Only Starts the Windows 98 operating system without loading Himem.sys, Ifshlp.sys, Config.sys, and Autoexec.bat, and does not load protected-mode Windows 98.You can also start this option by pressing ALT+F5.
Previous version of MS-DOS Starts the version of MS-DOS previously installed on this computer. You also can start this option by pressing F4. This option is only available if BootMulti=1 is set in Msdos.sys.

Booting Windows 98 in Safe Mode

A safe mode startup loads only the mouse, keyboard, standard VGA, and other drivers that Device Manager needs to function. Use safe mode when a computer does not boot because of incorrect drivers, corrupted files, or resource conflicts. With a minimal configuration, you can boot Windows 98 and reconfigure the system from the System icon in Control Panel. You can also use safe mode to prevent drivers in Autoexec.bat and Config.sys from loading.

When you select Safe Mode from the Windows Startup Menu, Windows 98:

  • Displays the message, "Windows is bypassing your startup files."

  • Bypasses the registry, Config.sys, and Autoexec.bat.

  • Loads Himem.sys and Ifshlp.sys.

  • Uses the file Win.com with the switch /d:m to load Windows 98.

Win /d:m

When safe mode executes Win /d:m, the file System.cb (cb stands for "clean boot") runs instead of System.ini.

Just before Win.com loads the shell (Explorer.exe), Win.com discards System.cb and loads System.ini. This enables Windows 98 to process settings in System.ini, except for the [boot] or [386enh] sections.

Tip

If safe mode eliminates a problem, examine the device entries in the [boot] or [386enh] sections of System.ini. A file in one of these sections may be causing the problem.

Windows 98 will not run any entries on the load= or start= lines of Win.ini or any entries in the Startup folder.

Troubleshooting in Safe Mode

If the computer boots into safe mode but not in normal mode, the majority of probable causes fall into one or more of these four major areas:

Problem Area Suggested Action
Network configuration Disable the protected mode networking drivers with Device Manager, and restart the computer. If this eliminates the problem, then the problem was that the network configuration was failing to load.
Protected mode disk driver Disable the protected-mode disk driver with Device Manager, and restart the computer. If this works, then the disk driver was failing to load.
Video driver Switch to the standard VGA driver. If this works, the display driver or its settings are incompatible with your system.
Third party VxDs Remark out the entry for a third-party VxD in System.ini, and restart the computer. Continue with this procedure for all third-party VxDs until you find the problematic driver.
