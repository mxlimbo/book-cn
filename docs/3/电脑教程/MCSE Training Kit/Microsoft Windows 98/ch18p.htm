Configuring Server Types and Protocols

To configure a Dial-Up Networking client, you choose the server type and protocol for the system into which you are dialing You can also configure advanced options that determine how you log on to the network, whether you use compression for data transmissions, and whether you require encrypted passwords. In addition, you can configure TCP/IP settings for each connection (see Figure 18.8).

Figure 18.8 Configuring TCP/IP for your Dial-Up Network Connection

If you want to configure Dial-Up Networking server types and protocols, perform the following procedure:

  1. From the Dial-Up Networking folder, right-click the connection you want to configure, and click Properties.

  2. On the Server Types tab, select the server type you want from the drop-down list box.

  3. Configure the Advanced options settings (see the table below for a definition of these settings).

  4. Click TCP/IP Settings to configure these settings.

  5. Click OK.

  6. Click OK.

The following table defines the Advanced options settings:

Select This Option To
Log on to network Log on to the network you are connecting to using the user name and password you used when you logged on to Windows 98.
Enable software compression Compress incoming and outgoing information before you send it. This helps speed up the transfer of information. Compression only occurs if both computers are using compatible compression.
Require encrypted password Specify that only encrypted passwords can be sent or accepted by your computer. This supplies additional security. The computer you are connecting to must support encrypted passwords for this option to take effect.
Require data encryption Specify that the computer will only connect to a server that uses data encryption.
Record a log file for this connection Create a log file that records information about the connection. Windows 98 writes the file modemname.log to the C:\Windows folder.

TCP/IP Settings

If you are using the TCP/IP protocol, you can configure TCP/IP settings for each connection. You can configure the following options:

  • Specify an IP Address: By default, your Windows 98 DUN client will receive a Server-assigned IP address. Windows NT Remote Access Service (RAS) servers can assign addresses, or they can relay the address request to a DHCP server. As an option, you can specify an IP address, which will always be used for that connection.

  • Specify Name Server Addresses: By default, your Windows 98 DUN client will receive Server-assigned name server addresses. Windows NT RAS servers can assign name server addresses by obtaining the information from a DHCP server. Optionally, you may choose to Specify name server addresses, up to two each for DNS and WINS.

You can also configure two optional parameters:

  • Use IP header compression: Improves performance by enabling negotiation and support of IP header compression. This option is enabled by default.

  • Use default gateway on remote network: Temporarily (as long as the connection is active) configures the RAS server as your client's default gateway, rather than the one configured in the Network option of Control Panel.
