Virtual Memory

Virtual memory is a term that refers to the fact that the operating system can actually allocate more memory than the computer physically contains. Physical memory is the amount of RAM and free disk space on your computer. Because physical memory is limited, Windows 98 allocates each process a unique virtual address space, which is a set of addresses available for the process's threads to use. This virtual address space appears to be 4 GB in size; 2 GB reserved for program storage and 2 GB reserved for system storage.

Figure 5.1 Windows 98 virtual memory structure

System and application components reside in virtual memory at various locations:

  • All Kernel-mode (Ring 0) components reside in the address space above 3 GB. Kernel-mode is the privileged mode of operation in which code has direct access to all hardware and all memory addresses.

  • The Windows 98 core components and shared dynamic-link libraries (DLLs) reside in the shared address space between 2 GB and 3 GB. This makes them available to all applications.

  • Win32-based applications reside between 4 MB and 2 GB. Each Win32-based application has its own address space to protect it from other applications.

  • Real-mode device drivers and terminate-and-stay-resident (TSR) programs reside at 640K and below.

Windows 98 Swap File

The virtual memory scheme requires memory to be swapped between RAM and the hard disk when the amount of free RAM becomes low. The file located on the hard disk where memory is swapped between RAM and the hard disk is called a swap file.

The Windows 98 virtual memory swap file implementation addresses the problems and limitations imposed in Windows 3.1x. Under Windows 3.1x, users had to decide whether to use a temporary swap file or a permanent swap file, how much memory to allocate to the swap file, and whether to use 32-bit disk access.

The swap file implementation in Windows 98 simplifies the configuration task for the user and combines the best of temporary and permanent swap files, due to improved algorithms and access methods. The swap file in Windows 98 is dynamic; it can shrink or grow, based on the operations that are performed on the system. The swap file can reside on a fragmented region of the hard drive or on a compressed volume with no substantial performance penalty.

If you want to change your swap file settings in Windows 98:

  1. From the Windows 98 Start menu, point to Settings, and click Control Panel.

  2. In Control Panel, double-click System.

  3. Click the Performance tab, and then click Virtual Memory.

On this property page, you can adjust the size, minimum and maximum settings, or let Windows 98 control the size of the swap file.
