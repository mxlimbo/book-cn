Configuring Logon Functions

When you configure logon functions, you must choose which network will validate your logon. You can configure some functions, such as the primary network logon and enabling logon script processing from a computer running Windows 98. However, you can change passwords only from a NetWare server.

If your users must have preferred servers authenticate them at system startup, configure the primary network logon to Client for NetWare Networks by performing the following procedure:

  1. In Control Panel, double-click the Network icon.

  2. Select Client for NetWare Networks and click Properties.

  3. On the General tab, choose a preferred server from the drop-down box.

  4. Select the first network drive from the drop-down box.

  5. Enable logon script processing is checked by default. If you do not want to have this function enabled, click the box to clear it.

  6. Click OK.

Changing NetWare Passwords

Many NetWare commands and utilities function only when run from the command line on a mapped drive; they do not function from a universal naming convention (UNC) path.

For example, you cannot change your password on a NetWare server using Windows Explorer. You must use the NetWare setpass utility from a mapped drive at a command prompt in order to change your NetWare password.

Using Logon Scripts

If you enable logon script processing in the Properties dialog box for Client for NetWare networks, Nwlsproc.exe downloads and executes the logon script from the preferred server. The program executes all commands except those for Terminate-and-Stay-Resident (TSR) programs.

Note

The logon script processor does not support NetWare 4.x-specific commands, and map or capture commands in the logon scripts cannot use NDS syntax.
