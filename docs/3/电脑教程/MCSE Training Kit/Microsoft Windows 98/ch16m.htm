Location of Policy Files

When a user logs on to a network with a computer running Windows 98, the operating system searches for a policy file, depending on the primary network logon, as shown in the following table:

Primary Network Logon Of Location Of Policy File Config.pol
Client for Microsoft Networks The Netlogon share of the primary domain controller for the user's domain. The Netlogon share is a default share on a Windows NT domain controller which corresponds to the C:\winnt\system32\repl\import\scripts folder on the server.
Client for NetWare Networks The SYS:PUBLIC directory on the preferred NetWare server.

Note

For group policies on a Novell NetWare network, at least one NetWare 4.1 server on the network must be running in bindery emulation mode. The group and all users in the group must be in bindery context for the server.

On large networks where many users can log on and attempt to download the same policy file within a short period of time, there is a potential for poor network performance. Windows 98 system policies offer two methods for managing this network traffic: load balancing and manual update.

Load Balancing System Policies

On a Windows NT network, users download the system policy from the Primary Domain Controller. To avoid a network slowdown on a Windows NT network, you can enable load balancing. Policies then can download from the logon server, which can be a backup domain controller rather than the Primary Domain Controller.

If you use load balancing, be sure that you maintain the policy file on all domain controllers in the domain. You can automate this process with the Windows NT Directory Replicator service.

If you want to enable load balancing, perform the following procedure:

  1. In System Policy Editor, open the \\computer_name\Windows 98 Network\Update\Remote Update policy.

  2. Select the Load balance setting.

Using Manual Update with System Policies

On NetWare networks, Windows NT networks, or in a stand-alone configuration, you can use the Manual Update option in the Remote Update policy to specify the location and name of the system policy file. Manual Update distributes the administration of system policies by applying different policy files to groups of users.

Note

The word manual in Manual Update refers to a defined location of the policy file other than the default location. The update still occurs when the user logs on, but Windows 98 searches for the file and path in the Manual Update setting, rather than the default location. When you use the Manual Update option, you can give the system policy file any 8.3 filename with the extension *.pol.

Although you can copy policy files to servers other than domain controllers and preferred servers with Manual Update, be sure to keep the original Config.pol file in the default location. New clients search the default location the first time they log on to the network. Subsequently, when the user logs on, Windows 98 uses the path and file name in the Manual Update setting.

System Policies in a stand-alone Configuration

You can also create policies for a stand-alone computer with the Manual Update option by directing Windows 98 to look in a local path for a policy file. This is useful when multiple users are using a stand-alone computer.

If you want to use system policies on a stand-alone computer, perform the following procedure:

  1. In System Policy Editor, click Open Registry on the File menu.

  2. Double-click the Local Computer icon, then the Windows 98 Network icon, and then double-click the Update icon.

  3. Select the Remote Update check box, and then click Manual in the Update Mode box. In the Path box, type the path and file name for your system policy file. You can place this file in any folder on the hard disk. Click OK.

  4. On the File menu, click Save.

  5. On the File menu, click New Policy. Select the system policy settings you want to use.

  6. On the File menu, click Save As. Save the file with the name and path you used in step 3.

  7. Exit System Policy Editor, and then restart Windows 98.
