What Are System Policies?

System policies are collections of registry settings that overwrite current user and local machine areas of the registry.

System policies are defined in a policy file (*.pol), usually named Config.pol. Windows 98 applies the applicable registry settings from the file to the computer's registry.

Important

The system policy must be named Config.pol if it is used in the default location for a net logon download.

A policy file may contain entries for the following:

  • Default user settings

  • Default computer settings

  • Settings for specific users

  • Settings for defined groups

  • Settings for specific computers

With system policies, you can do the following:

  • Restrict what users can do from the desktop.

  • Restrict access to Control Panel options.

  • Customize parts of the desktop.

  • Configure network settings.

For example, you can preset a user's environment so that the MS-DOS prompt or unapproved applications are not available.

You must use the Windows 98 System Policy Editor to create and modify system policies for computers running Windows 98.
