Optimizing CD-ROM File System Performance

The CD-ROM File System (CDFS) maintains its own internal cache in order to optimize access to the CD-ROM drive. Because CDFS uses a separate cache, the cache memory can swap to the hard disk when CD-ROM drive activity pauses. It is faster for Windows 98 to read data from the cache even if it has been paged to disk because the hard disk access time is much faster than the CD-ROM drive access time. This gives applications more room to run and protects the main hard disk cache from being emptied when a large amount of data is played back.

You set the CDFS cache with the CD-ROM tab of the File Systems Properties sheet (see Figure 6.6).

Figure 6.6 The CD-ROM tab of the File Systems Properties sheet

If you want to set the size of the CDFS cache:

  1. Click the System icon in Control Panel. Select the Performance tab and click File System. Click the CD-ROM tab.

  2. Choose the desired settings for Supplemental cache size and Optimize access pattern for.

    • Supplemental cache size controls how much additional memory is allocated to the CDFS cache for read-ahead caching. The supplemental cache for CDFS holds path, table, directory, and file information. The supplemental cache is allocated on a sliding scale from a minimum of 64K to a maximum of 1,238K.

    • Optimize access pattern for defines the minimum and maximum CDFS cache size, based on CD-ROM speed. Select the setting that matches speed of your CD-ROM drive. The following settings are available:

      • No read-ahead

      • Single-speed drives

      • Double-speed drives

      • Triple-speed drives

      • Quad-speed or higher
