Summary

The following information summarizes the key points in this chapter:

The Windows 98 System Architecture Overview

  • The core components of Windows 98 are the User component, Kernel component, the graphical design interface (GDI), and the user interface.

  • Key features of Windows 98 architecture include preemptive multitasking and multithreading support, a protected-mode file system that eliminates reliance upon MS-DOS once Windows 98 starts, and dynamic hardware and environment configuration.

  • For 16-bit applications, Windows 98 uses cooperative multitasking. For 32-bit applications, Window 98 uses preemptive multitasking, in which the operating system has complete control over running applications.

Windows 98 Memory Management

  • Using its virtual memory functions, Windows 98 can allocate more memory than the computer actually physically contains. Windows 98 accomplishes this task by swapping memory between RAM and the hard disk when the amount of free RAM becomes low.

  • Each process is allocated a unique virtual address space that is divided into pages. Code and data are moved in pages between physical memory and the swap file on the hard disk.

Virtual Machines

  • Win32- and Win16-based applications run within the System Virtual Machine. Win32-based applications each run in a separate virtual machine within the System Virtual Machine, while all Win16-based applications run together in a shared virtual machine within the System Virtual Machine.
