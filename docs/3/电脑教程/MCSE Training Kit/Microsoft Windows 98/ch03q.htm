How Windows 98 Generates MS-DOS File Names

Files in Windows 98 may have two file names: a long file name, and an alias, or MS-DOS name—a file name that conforms to the 8.3 character naming convention.

Figure 3.6 MS-DOS file name structure

An MS-DOS name (see Figure 3.6) consists of a basis name, a numeric tail, and an extension.

  • The basis name consists of the first six legal characters in the long file name, changed to uppercase. Legal characters include all alphanumeric characters and certain symbols. Spaces are not legal characters.

  • The numeric tail is a tilde (~) and a number, usually 1 (if the MS-DOS name is unique), which increases incrementally to avoid conflict with an existing MS-DOS name.

  • The extension consists of the first three legal characters after the final period in the long file name (LFNs can contain multiple periods).

For example, the LFN "This is a Long File Name.document.txt" has an MS-DOS name of "THISIS~1.TXT". If this MS-DOS name already exists for another file, the MS-DOS name would be "Thisis~2.txt"

Windows 98 continues to increase the numeric tail as necessary. After nine iterations, the basis name is reduced to the first five legal characters. The MS-DOS name in the above example would become "THISIS~1.TXT". This is unlike Windows NT, which adds hexadecimal characters to file names after the fourth iteration.

When working at a command prompt within Windows 98, you must either use the MS-DOS name or use quotation marks around the long file name. If you want to delete the above file, you would type: 

Del thisis~1.txt

– Or – 

Del "this is a long file name.document.txt"
