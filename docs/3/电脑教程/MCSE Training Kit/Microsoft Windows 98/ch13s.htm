Explicit vs. Implicit Permissions

In Windows 98 you can assign two types of user-level permissions: explicit and implicit. Explicit permissions are specifically granted to a user or group at some point in the shared folder hierarchy. Implicit permissions are inherited from the explicit permissions assigned to a parent folder. Figure 13.11 illustrates the difference between implicit and explicit permissions.

Figure 13.11 Implicit and explicit permissions

In a shared folder hierarchy, explicit permissions apply to all folders below the folder to which the permissions were assigned, unless different explicit permissions are assigned farther down the hierarchy. In this case, the new explicit permissions apply from that point downward through the hierarchy. This is unlike Windows NT, in which folder permissions cannot grant permissions not already granted through the share.

If you want to change permissions on a parent folder, perform the following procedure:

  1. Right-click on the shared resource and select Sharing.

  2. In the Change Security dialog box, choose one of the following options:

    • Apply these changes to inside folders: changes all explicit permissions applied to folders within this folder to match the permissions set at this folder.

    • Do not change any inside folders: leaves unchanged all explicit permissions applied to folders within this folder.

    • Display each folder name individually: prompts you to change the explicit permissions on each subfolder that has explicit permissions currently assigned.

You can change the permissions on a subfolder of a shared folder without sharing the subfolder. You can also create a separate share for a subfolder.
