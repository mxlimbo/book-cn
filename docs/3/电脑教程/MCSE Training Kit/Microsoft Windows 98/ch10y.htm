Using System File Checker

Troubleshooting system file problems can be a difficult task. Error messages that are generated due to missing or corrupt system files may be misleading and may not describe the problem properly.

System file problems can occur in several ways. For example, system file problems occur when:

  • Applications overwrite vital operating system files.

  • Users inadvertently delete vital files.

  • Disk errors corrupt system files.

System File Checker

System File Checker facilitates troubleshooting and solving system file problems. System File Checker scans operating system files looking for incorrect version numbers, file corruption, and missing files. If any of these problems are identified, System File Checker prompts you to restore the original files.

If you want to use System File Checker, perform the following procedure:

  1. Click Start and point to Programs, Accessories, System Tools, and then click System Information.

  2. On the Tools menu, click System File Checker.

  3. Click Scan for altered files, and then click Start.

The System File Checker scans the system using a file called Default.sfc.

The Default.sfc file contains the following information about each system file:

  • Cyclical Redundancy Check (CRC). Windows 98 adds a checksum to a file entry when it writes a file to disk. This data is used to see if the files have changed. For example, the sum of all binary ones and zeros in a file equals a specific number. If a CRC is performed on the file and the sum does not equal the checksum associated with the file, then the System File Checker determines that the file has changed.

    When System File Checker detects a difference between the information in the Default.sfc and the file on the disk, a message box appears. If all the verification data matches except for the CRC, the message states that there are differences in the contents of the file and recommends that you restore the file.

  • Date, time, size, and version information (if applicable). If System File Checker determines a change in date, time, size, or version, then the File Changed dialog box appears and displays the information for each changed file.

The following table describes the options that you can choose to update the file or the Default.sfc when System File Checker determines that a file changed.

Option Result
Update the verification information Update the file data in Default.sfc. Choose this option when you have updated a system file and want to update Default.sfc. System File Checker uses the updated information about the file the next time it is run.
Restore file Specify the source location of the file to restore, and where to save the new copy of the file. You are also prompted to back up the existing file to the C:\Windows\Helpdesk\SFC folder. You should choose to back up the file in case there are problems using the new file.
Ignore Skip the file. The next time System File Checker is run, this file causes the File Changed dialog box to appear because the file was not restored.
