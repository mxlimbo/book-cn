Using Templates for System Policies

System Policy Editor displays a template of a policy file as a graphical list of the settings you can control, with check boxes you mark to indicate whether that setting should be included in the policy or removed from the policy. Policy templates (*.adm files) determine the scope of a system policy file. The template controls which policy settings you can enable or modify.

Default Templates

When System Policy Editor starts, it loads the default policy templates (Common.adm and Windows.adm) from the C:\Windows\inf folder. You can use this template to create and modify a system policy file (see Figure 16.3), or you can open other templates through the Options menu in the System Policy Editor.

Several other templates are installed in the C:\Windows\inf folder, such as a policy file for controlling settings in Outlook Explorer (Oem.adm). Each template controls settings for different components of Windows 98. You can also create custom templates using a text editor.

Note

For instructions on creating custom templates, refer to the Windows 98 Resource Kit.

System Policy Editor opens templates in either Policy File Mode or Registry Mode. When you open a policy in Policy File Mode, you create and modify system policies for implementation on other computers. This is an indirect edit of the registry, and does not apply changes until users log on after you save the policy file and copy it to the appropriate network server locations. New Policy or Open Policy on the File menu opens a system policy in Policy File Mode.

When you open a policy in Registry Mode, you edit the registry of the local or a remote computer. This is a direct edit of the User.dat and System.dat files. Changes made to the registry with System Policy Editor do not take effect until you save the changes and close System Policy Editor.

Open Registry opens a system policy in Registry Mode for the local computer. Connect on the File menu opens a system policy in Registry Mode for a remote computer.

Caution

If a registry entry is defined in the template but not in the registry itself, System Policy Editor displays the default value from the template as if it were actually reading the value from the registry. There is no way to verify in System Policy Editor whether a value is in the registry or in the template.

Template Option States

The System Policy Editor lists the available settings from the template with check boxes for configuring each option.

Option State Means The Option Is
Box with check mark Selected. Implements this option in a policy, changing the state of the user's computer to conform to the policy when the user logs on. If the option was previously checked the last time the user logged on, Windows 98 makes no changes.
Shaded box Unavailable. The setting is unchanged from the last time the user logged on, and Windows 98 will make no related modifications to the system configuration.

The unavailable state ensures that Windows 98 provides quick processing at system startup because it does not need to process each entry each time a user logs on.

If a setting requires additional information, an edit control appears at the bottom of the Properties dialog box. For example, if you select Wallpaper in the Desktop Display settings, the controls to select the type of wallpaper and to set it as tiled appear.

Caution

If you set an option by selecting it, and then change your mind and clear the option, you can destroy the user's previous configuration. A cleared option removes the specified setting from the registry. If you decide not to set a particular policy option, make sure the check box is marked as "Unavailable" so that the configuration for that setting is unchanged.
