Installing and Configuring Dial-Up Server

Dial-Up Server provides single caller access to a Windows 98 computer through Dial-Up Networking. You must use either the Point-to-Point Protocol (PPP) or Asynchronous NetBEUI protocol and connect from a computer running one of the following operating systems:

  • Windows 98

  • Windows NT

  • Windows 95

  • Windows for Workgroups

  • Workgroup Add-on for MS-DOS

  • LAN Manager

Installing and Configuring a Dial-Up Server

Although Windows 98 automatically installs dial-up networking along with other communications tools when you set up the operating system, you must install Dial-Up Server manually (see Figure 18.11).

Figure 18.11 Configuring a Dial-Up Server

If you want to set up a Dial-Up Server, perform the following procedure:

  1. In Control Panel, double-click Add/Remove Programs.

  2. Click the Windows Setup tab.

  3. Select Communications, and then click Details.

  4. Click the Dial-Up Server listing to select it and click OK.

    After you install Dial-Up Server, you can configure it by choosing Dial-Up Server from the Connections menu of the Dial-Up Networking folder.

  5. From the Dial-Up Networking folder, open the Connections menu and select Dial-Up Server.

  6. Choose one of the following:

    • No caller access.

    • Allow caller access. If you choose this option, click Change Password to create or change a password for the connection.

  7. If desired, add a comment.

  8. To check or change the server type, click Server Type.

    On the Server Type tab, you can specify Type of Dial-Up Server, either PPP or Asynchronous NetBEUI (Windows for Workgroups and Windows NT 3.1) manually. Selecting Default allows the client and server to negotiate a protocol, attempting PPP first, then NetBEUI. PPP supports both NetBEUI and IPX/SPX-compatible protocols.

  9. You can also choose to enable software compression and require an encrypted password.

  10. Click OK, and then click OK again.

Note

Windows 98 Dial-Up Server does not support TCP/IP. To host remote connections to your intranet, use a Windows NT Remote Access Service (RAS) server.

Securing a Dial-Up Server

The type of security available for Dial-Up Networking depends on the type of access control used on the computer that has Dial-Up Server enabled.

On a computer with share-level access control, you can assign a password to Dial-Up Server. Any user who knows the password for the server will be able to connect to it.

If you are connected to a Windows NT domain controller or to a NetWare server that authenticates requests, you can use user-level access control. You can assign remote access permissions to specific users. The user must enter a user name and password as defined on your computer's security provider in order to establish the connection.

With both share-level and user-access controls, you can require encrypted passwords from clients. Encrypted password authentication uses the Challenge Handshake Authentication Protocol (CHAP). A handshake is a series of signals acknowledging that communication or the transfer of information can take place between computers or other devices. With CHAP, the server uses a handshake to verify the identity of the client without sending the password across the link.

A CHAP Logon

In a CHAP logon process, the server sends a random challenge message to the client. The client encrypts the challenge using its password as the encryption key and returns the challenge to the server. The server then compares the returned value with the sent value; a match means the remote user entered the correct user name and password combination and is allowed to log on. Because the challenge message changes each time a user logs on, and because the password itself is never sent over the link, CHAP is a safe method of user authentication.

Tip

Windows 98 Dial-Up Server does not support Multilink. Users who want Multilink capabilities for their dial-up servers must use Windows NT 4.0.
