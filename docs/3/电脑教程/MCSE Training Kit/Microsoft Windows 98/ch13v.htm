Lesson Summary

A computer running Windows 98 can connect to any computer running Windows NT and use that computer's file and print resources. To connect to a computer running Windows NT in a domain, you must log on to the domain with a valid domain user account and password, and your account must have access rights to the computer running Windows NT. To connect to a computer running Windows NT in a workgroup, you must enter a valid user name and password for the remote computer when you log on to your local Windows 98 computer.

To print from a computer running Windows 98 to a print server running Windows NT, the server must have Windows 98-compatible print drivers. You can administer a remote Windows NT printer the same way you administer a remote Windows 98 printer.

When you employ user-level access control on a Windows NT network, you must choose a security provider, which is a Windows NT domain server or a Windows NT workgroup computer that maintains the list of users and groups to which you assign resource permissions.

When you use user-level security to protect shared resources, you assign permissions to users and global groups by editing the Access Control List.
