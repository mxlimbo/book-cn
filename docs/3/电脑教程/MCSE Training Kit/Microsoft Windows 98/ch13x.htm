Summary

The following information summarizes the key points in this chapter:

Interoperating with Windows NT

  • A Windows 98 computer running on a Windows NT network can log on to a domain, share resources with a computer running Windows NT, gain access to Microsoft BackOffice applications running on a Windows NT Server, and download user profiles and system policies.

  • The Windows 98 Client for Microsoft Networks supports the Microsoft Distributed File System (DFS) for Windows NT Server, which provides a single tree structure for accessing multiple shared resources on a network.

Logging on to a Windows NT Domain

  • In Windows 98, you can log on to a Windows NT domain and connect to domain resources. You configure Windows 98 to log on to a Windows NT domain through Client for Microsoft Networks.

  • To create a logon script, use the User Manager for Domains tool on the Windows NT Server. You can use the Passwords option in Control Panel to synchronize user passwords, which gives users the ability to use a single password to log on to both Windows 98 and Windows NT.

Using Windows NT Network Resources

  • A computer running Windows 98 can connect to any computer running Windows NT and use that computer's file and print resources.

  • To print from a computer running Windows 98 to a print server running Windows NT, the server must have Windows 98-compatible print drivers. You can administer a remote Windows NT printer the same way you administer a remote Windows 98 printer.

  • When you employ user-level access control on a Windows NT network, you must choose a security provider. A security provider is a Windows NT domain server or a Windows NT workgroup computer that maintains the list of users and groups to which you assign resource permissions.

  • When you use user-level security to protect shared resources, you assign permissions to users and global groups by editing the Access Control List.
