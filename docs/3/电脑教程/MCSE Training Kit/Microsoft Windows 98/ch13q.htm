Editing an Access Control List

When you employ user-level security to protect shared resources, the users and global groups that are permitted to access the shared resource appear on the Access Control List (ACL).

Editing the Access Control List

You can add and/or remove users and groups from the ACL by clicking the Add and Remove buttons.

Figure 13.9 Editing the Access Control List with the Add Users dialog box

If you want to view and edit the Access Control List, perform the following procedure:

  1. Right-click the shared resource.

  2. On the Sharing tab, click Add.

    The Add Users dialog box displays a list of global groups and users available on your security provider.

  3. If your security provider is a Windows NT domain server, you can also assign permissions to users in trusted domains, by selecting the users' domains in the Obtain list from list (see Figure 13.9).

User vs. Group Permissions

If the ACL grants permissions to both a user and a group to which the user belongs, the user permissions override the group permissions. This is unlike Windows NT, in which user and group permissions are cumulative. In general, you should assign permissions to groups rather than to users, and then assign users to the appropriate groups.

Important

To create groups or to assign users to groups you must be a Windows NT Domain Administrator.

The World Account

If you do not want to specify individual users and groups on a low-security share, you can enable public access by granting permission to The World. The World account grants access to any valid user on the security provider. If the security provider's guest account is enabled, then The World account grants access to users outside the domain as well.
