How Windows 98 Generates Long File Names

Windows 98 implements long file names on FAT partitions by concatenating a series of MS-DOS-compatible directory entries into a single file name entry.

To mask these directory entries from MS-DOS, Windows 98 assigns the following attributes to the entries:

  • Read-only

  • System

  • Hidden

  • Volume Label

Because MS-DOS does not recognize this combination of attributes, it ignores these directory entries.

Windows 98 supports a maximum of 255 characters in a long file name, but the entire path to the file may not exceed 260 characters.

If you open and save a file with a long file name in a program that does not support long file names, Windows 98 preserves the long file name as long as you do not rename or move the file.

Windows 98 preserves long file names on a network with a process called tunneling. Tunneling preserves long file names for files that you open and save with an application that does not recognize long file names, on both the local computer and on the network. In order to perform network tunneling, the network operating system running on the server must support long file names.
