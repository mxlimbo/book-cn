Enabling Deferred Printing

You can configure printing to defer it until a later time. With deferred printing you can store a print job on your computer's local hard disk. You can print the document later without opening the application. This function is useful in at least two situations:

  • You are using a mobile computer and are working away from your docking station to which a printer is attached.

  • You are using a computer on a network, and the remote printer is malfunctioning or is unavailable.

Unavailable printers appear in the Printers folder as grey icons. You can still send print jobs to the unavailable printer, and documents sent to the printer are still spooled. When the printer becomes available, the spooled documents will print.

Note

Deferred printing files are stored in the %windir%\spool\printers folder with .spl extensions. In the case of EMF spooling, the .spl file contains pointers to the EMF page files located in the temp folder.

Configuring Deferred Printing

Deferred printing is automatic for a laptop computer that uses a docking station. While the laptop is undocked, deferred print jobs are stored in its printer queues. After you dock the laptop, Windows 98 prompts you to print the deferred print jobs.

You can also manually configure a print queue to use deferred printing by selecting the Work Offline option on the File menu in the Printers folder. When you are ready to print you must deselect the option manually before Windows 98 will prompt you to print the deferred print jobs.
