Optional Practice: Logging on to a Windows NT Domain

In this practice, you will log on to a Windows NT domain.

Answers to the questions in this practice can be found in Appendix A, "Questions and Answers" located at the back of this book.

    To test access to domain resources if you do not have domain validation

    1. Click Start, and then click Run.

      The Run dialog box appears.

    2. In the Open list box, type \\ntserver\domonly and then click OK.

      Notice that the Enter Network Password dialog box appears. Windows 98 displays this dialog box when you attempt to connect to a shared folder in a domain in which you do not have a valid domain user account.

    3. Click Cancel to close the Enter Network Password dialog box.

    4. Click Cancel to close the Run dialog box.

    To configure the Client for Microsoft Networks to log on to a Windows NT Domain

    1. In Control Panel, double-click Network.

      The Network dialog box appears.

    2. In the list of installed network components, click Client for Microsoft Networks, and then click Properties.

      The Client for Microsoft Networks Properties dialog box appears.

    3. Select the Log on to a Windows NT domain check box.

    4. In the Windows NT domain box, type your domain name.

    5. Click OK to close the Client for Microsoft Networks Properties dialog box, and then click OK again to close the Network dialog box.

      A dialog box appears prompting you to restart your computer.

    6. Click Yes.

      Your computer restarts.

    To test access to domain resources when you have domain validation

    1. What has changed in the Enter Network Password dialog box?

      Answer

    2. Enter your Domain_Username and Domain_Password, and then click OK.

      Answer

    3. Press any key to continue the logon script.

      The Set Windows Password dialog box appears.

    4. In the Confirm new password box, type your Domain_Password and then click OK.

      You will be logged on to the Windows NT domain.
