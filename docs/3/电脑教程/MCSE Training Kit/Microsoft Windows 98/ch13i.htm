Configuring a Domain Logon

You configure Windows 98 to log on to a Windows NT domain through Client for Microsoft Networks (see Figure 13.4).

Figure 13.4 Client for Microsoft Networks Properties sheet

To configure your computer to log on to a Windows NT domain, follow these steps:

  1. In Control Panel, double-click Network.

  2. Select Client for Microsoft Networks and click Properties.

  3. In the Client for Microsoft Networks Properties dialog box, select the Log on to Windows NT domain check box, and type the name of your domain in the Windows NT domain box.

  4. You can then select whether to use Quick Logon or Logon and restore network connections.

    • Select Quick logon to log on to the network without reconnecting any previously used network drives until you attempt to use them (this makes the logon process faster).

    • Select Logon and restore network connections to have Windows 98 attempt to restore all network connections during the logon process. This slows down the logon process but it also lets you know when you log on whether or not network resources are available.

Before you continue with the lesson, run the Chap13.exe animation located in the Animations folder on the Supplemental Course Materials CD-ROM that accompanies this book. This animation demonstrates how to configure your computer to log on to a Windows NT domain.

The Logon Process

When you start a computer running Windows 98, a logon prompt appears before you can access the desktop. Whether you are prompted for the local Windows 98 password or your domain password depends on the Primary Network Logon setting you have designated using the Network icon of Control Panel.

  • Windows Logon prompts you for your username and password for the local computer. This information is stored in a Password List (.pwl) file located in the C:\Windows folder. After the desktop appears, if you attempt to access a network resource, your local username and password are sent to the remote server.

  • Client for Microsoft Networks prompts you for your domain name, domain username, and domain password. This information is stored on the domain controllers in your Windows NT domain. If your domain username and password match your local username and password, your desktop appears after you log on to the domain. Otherwise, the local Windows Logon dialog box appears, prompting you for your local username and password.

  • Microsoft Family Logon prompts you to choose your user name from a list of names stored on the computer. After you click your user name, you can enter your password. You are not prompted for a domain name; however, if the computer is configured to log on to a domain, and your domain username and password do not match, you are prompted for a Microsoft Network logon.

To use Microsoft Family Logon, you must have user profiles enabled. To log on to a network using Microsoft Family Logon, you must have network client software, such as Client for Microsoft Networks, installed.

Logon Scripts

You can use the User Manager for Domains tool on Windows NT Server to assign a logon script to a Windows 98 user. A logon script is a batch file that runs automatically when a user logs on, before the desktop is displayed. Typical logon scripts will include commands to customize a user's computer, such as setting the time or connecting network drives.

When a Windows 98 user logs on to a Windows NT domain, the user's logon script (if one is assigned) is downloaded from the netlogon share on the authenticating domain controller. Lmscript.exe and Lmscript.pif, which are located in the C:\Windows\System folder, generate the process that executes the logon script.
