Printing from MS-DOS Programs

MS-DOS applications do not benefit from EMF spooling, because they do not use the GDI for their printing process. Therefore, an MS-DOS print job may take longer to finish than a Windows print job does using EMF spooling. However, MS-DOS applications can use the Windows 98 spooler and can, therefore, benefit from background printing.

Note

Print spooling for MS-DOS applications works only if the printer is installed to an LPTx (parallel) port. MS-DOS print jobs for COM (serial) ports cannot spool, because Windows 98 cannot distinguish a print job from other types of MS-DOS application serial output.

Disabling MS-DOS Print Job Spooling

MS-DOS print job spooling is enabled by default. There are instances in which spooling of MS-DOS print jobs is automatically disabled:

  • If a particular port is redirected to a network printer, MS-DOS spooling is disabled for that port. This supersedes the Spool MS-DOS print jobs option for the printer.

  • If Windows 98 is running in MS-DOS mode, the entire Windows 98 spooler is unavailable, so MS-DOS print job spooling is unavailable.

You can disable MS-DOS print job spooling manually in the printer's property dialog box by clicking Port Settings on the Details tab and clear the Spool MS-DOS print jobs setting.
