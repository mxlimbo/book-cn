<A NAME="367"><H1>Lesson 4: Managing Software</H1></A>

<p>Without the management controls built into Software Installation,
the value of having users automatically install software would have to
be weighed against potential software installation hazards. Those
hazards could include the following:</p>

<ul>
<p><li>Users encountering new file extensions (such as .GIF) that
could install a number of different applications (such as Adobe
Photoshop or Microsoft Internet Explorer)</li></p>

<p><li>Users installing every program listed in Add/Remove
Programs, whether they actually needed the programs or not</li></p>

<p><li>Users being reassigned to positions that precluded the use
of software that was installed for use only in their previous
positions</li></p>
</ul>


<p>To address these problems, Software Installation includes a number
of options for managing software after it has been deployed. In
particular, administrators have the ability to do the following:</p>

<ul>
<p><li>Associate file extensions with applications</li></p>

<p><li>Prevent software from being installed through document
invocation</li></p>

<p><li>Control the programs listed in Add/Remove Programs in
Control Panel</li></p>

<p><li>Categorize programs listed in Add/Remove Programs in Control
Panel</li></p>

<p><li>Automatically uninstall software whenever a GPO no longer
applies to an individual user</li></p>
</ul>

<A NAME="368"><H2>Associating File Extensions with Applications</H2></A>

<p>Active Directory directory services include a list of file
extensions and the applications associated with those extensions. This
is the list used by the Windows Installer whenever a user double-clicks
an unknown file type. System administrators cannot dictate the contents
of this list, but they can determine the priority for installing
applications upon document invocation.</p>
