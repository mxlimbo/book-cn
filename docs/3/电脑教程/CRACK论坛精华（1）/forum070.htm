        README.TXT

一、文件列表


    Locking.dll    判断软件是否注册并提供注册等的动态链接库，

    Locking.vxd    虚拟设备驱动程序, 请拷入 Windows9X 的 system 目录中，NT 中不需要。

    Register.exe    32位判断软件是否注册并注册的程序,
            运行环境: Windows2000、WindowsNT、Windows98、Windows95

    MkReg.exe    注册码产生软件（网上提供该软件演示版。需试用注册密码者，
            请把您的软件序列号寄来，换取注册密码）


二、Locking.dll 提供的函数等

    int Testkey(char * testlock);
                //检测软件是否注册并提供注册的函数
                //已注册或注册密码正确返回值为 0
                //没注册但软件使用天数不多于 15 天，返回值也为-2。
                //可根据该值设定软件只能运行部分功能。
                //其余情况返回值为 -1。

                //在调用Testkey(testlock)前，要定义 char testlock[10];    
                //软件已注册或没注册但软件使用天数不多于 15 天，
                //此数组各元素的值分别
                //为 8，否则为随机数。
                //在应用程序中调用 Testkey(testlock) 后，穿插
                //使用或判断testlock数组，能有效防止程序被软
                //解密。


三、软件模块的建议使用方法


1、在您的软件中调用检测软件是否注册的函数 Testkey( )，返回值为零，说明软件已注册，软件可继续运行；否则软件没有注册，应中断您的软件。建议在软件刚启动时就调用这个函数。

2、在调用检测软件是否注册的函数 Testkey( ) 后，穿插使用或判断数组 testlock[10]，能有效防止程序被软解密。

3、软件最终用户需把软件运行时提供的序列号，寄给软件出版商换取注册密码。

    在打包您的软件产品时，要同时把 Locking.vxd 打包进去，在安装软件时，要首先把它拷入 Windows9X 的 system 目录中。


四、本加密软件特点

1、使用本注册密码注册加密的软件，用户只能在一台机器上安装使用。把软件拷贝到其它机器上不能运行。

2、若用户想在另一机器上安装运行，必须把软件在这一机器上运行时的序列号，寄给软件出版商换取注册密码。当然应再交一份软件费用。

3、本加密方法特别适应在因特网上发布的软件及用光盘发布的软件。
