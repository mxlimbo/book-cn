第一页：优化

既然你已经知道了做动画GIF的一些简单方式，现在可以学习如何使它们值得在网上发布了。你很幸运，我们已为你做了痛苦的优化探索工作。每种创建GIF89的方式都有它自己的目标。在你开始确定使用哪种方式之前仔细读下面的信息。

祯优化

做动画时要很好地利用祯优化。让每个动作在物理上很接近。既然你在覆盖前一祯的同时想使文件尽可能的小，覆盖祯应很小。


例子：Webmonkey Toolkit Splash
动画制作：Judd Vetrone
媒体：GIF89
文件大小：40KB

注意猴子是如何在奔向工具箱之间做小的、近距离运动的。你根本注意不到运动的缺陷，只有运动的拆解。就应该这样。

通过选择Options:Frame Optimization，你可以自动修剪无用的或多余的祯。GifBuilder的这个功能不是一直好用，你的应用程序甚至可能不包括这个特征，所以你可能不得不手工优化这些祯。手工优化包括在
图像编辑程序中修剪祯，然后把它们拖放到GIF89程序中，然后手工地为它们确定位置。听起来有点难。关键是前一祯要被下一祯覆盖。感到困惑了吗？试几次就掌握了。

为什么要这么麻烦？因为像素越少，文件越小。再加上这种方法趋向于减少每祯的颜色数，实际上整个文件的大小也随之减少了。

透明图像

为了应用透明技术，用一个背景图像作为动画的第一祯，然后把透明的祯放在背景上。每幅透明祯都在一个或一些区域有动作（或图像数据），但是它们的其余部分是实色 - 可以在GIF动画程序中指定为透明色。有点令人困惑，但是在下载完下面的例子并在GifBuilder和Photoshop中打开后，你就会明白了。


例子：Absolut Panuska Map Art
动画制作：Lorelei Pepi
媒体：GIF89
文件大小：140KB

覆盖

当你用AfterShock把一个Macromedia Flash文件输出为GIF89文件后，程序会产生利用透明和祯优化的精致的祯。在这个例子中，你会看到黑色背景最终在GIF89中被渲染成透明，祯覆盖的只是前面运动中的像素。如果你已经在Photoshop中作好层，这样就可以覆盖前一祯的动作。当然结构文件会小一些。


例子：Frame from AfterShock exported GIF89
动画制作：Fork Unstable Media
媒体：AfterShock exported GIF89
