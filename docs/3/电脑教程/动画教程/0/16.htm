第一页 动画教程 - 第四天

当Netcscape 2.0的第一个插件出现时，Web上才真正出现动画。人们对终于有能动的东西而高兴，但是还能听到抱怨。“我们希望能索引！我们希望能查询！”商人尖叫着。“我们想看到源代码！我们希望能做快速容易的修改！”开发者热望着。“当我想看什么时，我不想下载一个
2MB的插件！”上网者呜咽着。

于是，dHTML诞生了，一个具有JavaScript，层叠样式表和HTML组件的作法自毙的语言。它看起来是对这些抱怨的完美的答复。商人得到他们的索引，开发者看到源代码，公众不必再担心不必要的下载。

但是并非完美无缺。你看，你可以用dHTML做各种精彩的事，但是从技术和创造性角度来说，Netscape和 Microsoft的浏览器在处理dHTML方式的巨大的差异使 dHTML的开发者面临一场噩梦。技术由于太好而不能
用，但是很少有人开发跨浏览器，跨平台的dHTML。

然后Macromedia出现了，它可以减少一些开发dHTML的痛苦。

Macromedia因为Director而出名，一个多媒体制作系统，回过头来使CD-ROM们愤怒。当互联网在1995年开始火起来的时候，Macromedia为Director发布了Shockwave插件，使网上漫游者在线观看Director的
内容。然后Macromedia又使Flash横空出世。当公司还在均衡这些插件时，它已经开始理解dHTML的重要性了。为了满足dHTML开发团队的增长需求，Macromedia发布了一个绝对好的产品 - 不仅引人注目地加速了HTML产品，而且使制作动态网页更容易。这个产品叫Dreamweaver，你可以看出来，我很喜欢它。它有缺点，但是对你的帮助却非常大。
