第一页：用现有条件做得最好

当以今天的标准进行衡量时，Web动画和制作工具看起来有些原始，但是别忘了Web只有五年的历史。把今天年幼的Web动画技术与有四十年历史的传统媒体相比，动画工具显得粗糙。

动画工具有其局限性，但也别对它们全面否定。到现在为止，某些动画技术和方式还在你能达到的范围之外。在你的动画中填充很多信息，你会受到带宽或处理器速度的限制，所以应该尽量裁剪你的动画，而不是相反。关键是把Web的劣势转化为你的优势。

我们前面提过，Monty Python的动画制作者和South Park的制作者Terry Gilliam喜欢用的“挖剪切图方式”（cutout style）在Web限制内工作得很好。而且这种方式与dHTML动画有限的运动方式很匹配。关键是保持运动简单。把一个基本的动画放在前景上，比如说，一个人在单车上的循环动画，但是位置不变。想使它看起来在地面上运动，可以使背景滚动。更好的方式是，建立一个小的（在尺寸和文件大小上）背景图像，然后把它扩展到最终大小。这很大地减小了文件大小，而图像质量没有改变。
