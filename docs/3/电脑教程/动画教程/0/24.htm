第五页：要记住的其它事情

我以前说过，并且现在再说一遍：“浏览器生而不平等”。当跨平台工作时，密切注意每种浏览器都支持什么。在Dreamweaver中起作用的东西不一定在浏览器中起作用，所以要尽可能在不同的机器、不同的平台和浏览器上测试你的动画。你可能发现你自己机器上的美人在你邻居的机器上变成张飞。或许你会发现你在PC上创建的令人惊异的工作会在Mac上崩溃。同时，查看不同平台或浏览器上是否有gamma或颜色问题。不同机器上颜色的不同会使设计者愤怒。你可能不希望知道不同浏览器和平台之间的细小差异，但是你用dHTML越多，你就会越好地避免这些缺陷。

有些内容很难转换成dHTML。或者你的动画太复杂，或者交互性需要太多的JavaScript。在这种情况下，可以考虑其它动画媒体类型，如Flash，GIF89或可按比例缩减的dHTML动画。这对令人灰心的浏览器也是一个好的方案。

另一个恼人之处是你的观众不仅用不同的浏览器，也用浏览器的不同版本。实际上，一些懒人用的浏览器太旧，以至于他们根本看不到dHTML。容纳这些旧浏览器的最好方式是提供另一种动画媒体类型，通常是可以普遍看到的GIF89。但是别为重建dHTML动画感到压力。用替代的媒体方式在解决麻烦上还是值得的。

IE 4.0提供了一种全屏幕方式，当适当地使用时，这种表现形式很有效。大多数CD-ROM使用全屏方式显示它们的内容，但这对Web用户是一个相对新的概念，而且可能令人迷惑。在观众下一次点击之前让他们知道后面的内容，或者为用户提供全屏选项。

你绝不会知道人们设置的他们的浏览器的大小 - 这涉及到个人爱好。考虑用百分率为你的“演员”定位，而不是用固定的像素尺寸。使用dHTML，有可能使你的动画应用所有可用的资源。那么，为什么不必要地限制你自己呢？

现在，你已经掌握了一些做dHTML动画的规范。这些指南只是基于我自己的经验。不同的人有不同的工作方式，并且它们都有效。如果你想在一个160*160像素的方块中演示动画，就把它放在框子里。如果你想让动画有更多的交互性，应该再做些什么。Web的很大一部分仍未被定义，dHTML有更多的事要做。人们还象以往那样行事，你也该这么做。谁知道呢？你想得到的可能是真正动态的东西。
