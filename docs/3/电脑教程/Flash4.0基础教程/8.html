　　 TRACE BITMAP：描绘位图

　　 这个功能能够将导入的位图转换成向量图，这样文件就小多了。：）不过Trace Bitmap命令只能够将位图转换成带有不连续色彩区域的矢量图形，效果上不如位图。哎，美中不足。

　　 如果你要转换位图，选中当前场景中的位图，选择Modify->Trace Bitmap，再根据你要求的质量进行参数设置即可。相关的参数介绍如下：

　　 （1）COLOR：输入Color Threshold值，输入范围为0-500，当对二个像素进行比较的时候，如果RGB的颜色值小于极限值，二个像素可以看作同一颜色。极限值越低，你所得到的颜色越多，效果也越接近。
　　（2）Minimum Area：设定一个像素周围所涉及到得像素数，值越小越精确。
　　（3）Curve Fit：确定缩绘制轮廓线的平滑度。
　　（4）Corner：确定时保留锐化边缘，还是进行平滑处理。 一般情况下，以下设定比较接近原始位图：

　　 Color：10、Minimum Area：1 pixel、Curve Fit：Pixels、Corner Threshold、Many Corners。

　　 Align：排列对象
　　 Align这个命令可以将多个选定项目在场景中进行排列。 排列时可以对元素尺寸进行修改以便它们的水平或垂直尺寸保持相同。 对元素尺寸的修改是满足最大的选定项目的尺寸， 排列命令还可以重新分布元素，使得其中心，左边或上边界， 右边或下边界间隔均匀。 排列选项可以单独使用或组合使用。Align to Page选项可以一次进行相互之间的排列和同舞台之间的关系排列。 当排列几个对象时， Flash会将其沿着将所有对象都封闭起来的矩形框的边缘或中心进行排列。 这和你选取对象的顺序无关。

　　 GROUP：群组
　　 使用这个命令你可以将多个对象结合起来，这样便于选取、编辑、移动。 编辑群组：选择Edit->Edit Selected或用选取工具双击群组对象，这时其他对象处于非激活状态，不可以被编辑和选择，双击群组以外的区域群组对象就恢复为一体，其他对象也可以被编辑了。

　　 UNGROUP：取消群组

　　 BREAK APART：打散
　　 这个命令用来将群组、文本、位图、INSTANCES及其他程序的OLE对象分成独立的可编辑的元素。 它可以显著地减小输入图形的文件尺寸。 打散命令完全不可逆转，并且对对象有下述影响：

  • 切断实体同其符号的联系。
  • 在动画符号中会放弃除当前帧外的所有帧。
  • 将位图转换成填色。
  • 将文本字符转换成轮廓线。
  • 切断OLE对象同其原创作程序的联系。
