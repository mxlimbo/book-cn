Lab: Using Visual Component Manager

In this lab, you will add an ActiveX DLL project to VCM.

To see a demonstration of this lab, run the Lab01.exe animation located in the Animations folder on the Supplemental Course Materials CD-ROM that accompanies this book.

Estimated lesson time: 45 minutes

  • To open Visual Component Manager

    1. On the Visual Basic Add-Ins menu, click Add-In Manager.

    2. In the Available Add-Ins list, scroll down and highlight Visual Component Manager.

    3. Under Load Behavior, select the Load on Startup check box.

    4. Click OK to add Visual Component Manager to the Visual Basic toolbar.

    5. From the View menu, click Visual Component Manager.

      6. Visual Component Manager opens.

  • To add a component to Visual Component Manager

    1. Click the COM Servers folder, and click Publish a New Component, as illustrated in Figure 1.9.

      2. Figure 1.9 The Visual Component Manager

      The Visual Component Manager Publish Wizard opens.

    2. For the component name, type Lab1-Sample.

    3. For the primary filename, type or browse to C:\Labs\Lab01\Component\In32_svr.dll.

    4. For the Type, click COM Server Library.

    5. Type your name as the Author.

    6. In the Component Includes category, select Source Code and Documentation and then click Next.

    7. Type a description and select ActiveX as a keyword, then click Next.

    8. In the Select Additional File(s) dialog box, click Add Files.

    9. Add the In32_svr.exp and In32_svr.lib files.

    10. In the Select Files that Require COM Registration dialog box, select In32_svr.dll and click Next.

      11. Your component is now published in the local database, as illustrated in Figure 1.10.
