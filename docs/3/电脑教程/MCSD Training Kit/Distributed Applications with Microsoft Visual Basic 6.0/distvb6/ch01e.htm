Summary

The Microsoft Solutions Framework (MSF) is a development framework produced by Microsoft Consulting Services (MCS). It is based on Microsoft's internal development methodology, but also includes the practices of hundreds of MCS corporate clients. Using the MSF can help you to create applications that deploy processes and data in a network environment.

In addition, Visual Basic 6.0 Enterprise Edition is designed for teams of developers creating distributed, high-performance client/server, Internet, or intranet applications. By developing with Visual Basic 6.0 Enterprise Edition, you can take advantage of the Windows NT platform and provide native code applications for systems that run on Windows 95 and Windows NT. Use the following tools and utilities to effectively develop and deploy your distributed applications:

  • Windows NT Service Pack 4 (SP4)

  • Microsoft Transaction Server

  • SQL Server

  • DCOM 98

  • Visual Source Safe (VSS)

Furthermore, the system architecture is an important feature in application design. The system architecture defines how elements in the application interact and what functionality each element provides. The following are three types of system (application) architecture:

  • Single-tier (or monolithic)

  • Two-tier

  • Multi-tier (or n-tier)

When developing a distributed n-tier system, you can build intelligent clients with business rules that have been compiled into stand-alone DLLs. These DLLs can be written in Visual Basic and reside on a server, most likely in Microsoft Transaction Server. COM and DCOM are the technologies that enable these DLLs (components) to communicate with each other.

Microsoft Visual Database Tools can also be used for building data-centric components. The Microsoft Visual Database Tools help you create and modify database objects. For example, you can create Microsoft SQL Server and Oracle database tables, use drag-and-drop operations to create views, and automatically change column data types.
