Summary

Overview of Universal Data Access

Universal Data Access (UDA) is a strategy for providing access to all types of information across the enterprise. OLE DB is the low-level interface that realizes UDA. The components of OLE DB are data consumers, data providers, and services. Microsoft ActiveX Data Objects (ADO) is the high-level, easy-to-use application interface that simplifies development against OLE DB data sources. ADO is the data-access method Microsoft recommends for new applications.

Introduction to the ADO Object Model

The ADO object model has seven objects: Connection, Command, Recordset, Field, Parameter, Error, and Property. Use the ADO Connection object to establish a connection to a data source. Use Command objects to specify the data to retrieve specified data. Command objects can be based on either a database or a SQL query. Recordset objects are constructed using records from a data source. Use the Recordset object to manipulate data from a provider. With ADO, you work with data almost entirely using Recordset objects.

Using the Data Environment

The Data Environment is a Visual Basic designer that provides an interactive, design-time environment for creating objects that access data. Add a Connection to a Data Environment to select a data source provider, and specify connection properties such as the database server name, username, password, and database name. Use the Data Environment designer to create Command objects to access data from a data source specified in the Connection object.

You can also access the Query Designer from the Command Properties window. Use the Query Designer to drag and drop database objects from the Data View window to build SQL statements. Drag and drop the Command from the Data Environment onto a form to easily design a data bound form.
