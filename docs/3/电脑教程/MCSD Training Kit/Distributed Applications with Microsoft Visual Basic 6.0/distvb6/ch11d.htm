Summary

Making coding mistakes that result in bugs is a part of developing applications. Visual Basic's debugging tools help you fix these problems. You can use the Immediate window, the Locals window, and the Watch window when you are debugging your application. The Immediate and Locals window allow you to evaluate expressions, change the value of variables, or execute different procedures. The Watch window allows you to specify logical expressions or variables that you want Visual Basic to monitor. Based on your settings, Visual Basic will take action when the variable changes or when the expression evaluates to True.

Unhandled run-time errors can cause your application to terminate and potentially cause the user to lose data. You should create error handlers in each event and general procedure in your applications. Errors can be handled using the On Error GoTo statement, or they can be handled inline. When you test COM components in Visual Basic, you either create a project group or run the component and test client projects in separate instances of the IDE. You can also use Visual Basic debugging facilities to test and debug MTS components. Visual Basic will automatically call into MTS to run your component in the MTS run-time environment.
