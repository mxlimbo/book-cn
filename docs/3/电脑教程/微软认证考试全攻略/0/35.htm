1) One PC has TCP, IPX, NetBEUI, and client for NetWare, File printing service installed. How to increase the Network performance.

a. Set the file property to server mode
b. Install file-printing service for Microsoft.
c. Unbind TCPIP for NW.
d. Unbind all unused protocol.

Answer mentioned was B c d

My opinion:
Is option "a" wrong. Server mode is supposed to improve preformance. Is the 2nd option file-print service for Microsoft better than File-print service for Netware. Is the option "b" correct. Reg the other two options, ( c and d ) I feel they are right.

Answers are a and d
A is correct because it increases disk cache says and means less reads to the hard drive and thus better performance.
B is incorrect because it is impossible to run both file and print services for Microsoft while running file and print services for NetWare
C is incorrect because TCP/IP is not bound to client for NetWare so this would be impossible--On the test the computer in question also ran client for Microsoft.
D is correct because extra protocols slow things down.


2) A computer named Y has 2 Network Interface Cards installed it. Y is connected to a network that includes 2 Novell servers. Y is running File and Print sharing for Novell Networks, IPX/SPX, and TCP/IP. Y is also connected to a smaller network that has 2 computers, Q and X. Q is a DOS client running a Novell NetWare Client and X is a Windows 95 computer running Microsoft Client for Novell Networks. What computers can Q and X see?

a) Q and X can see the resources shared on Y and make use of the resources stored on the Novell servers
b) X can see the resourcesshared on Y and make use of the resources stored on the Novell servers. Q cannot see any resources.
c) Only Y can use the resources of the Novell servers. Q and X cannot access any resources on either network.
d) X can see and use the resources on Y, however cannot make use of the resources on the Novell servers. Q cannot use any of the resources.

My opinion:
Win 95 doen't do routing. I feel the answer is option "d". The computer X can see Y, but the machine Q cannot see due to SAP advertising OFF ( not mentioned in question ).

I had a similar question but in my scenario the Q and X were both using a NetWare Client (not windows client for NW) Because of the other answers being impossible on my test--again answers slightly different than these, I had to assume that SAP was being used. The only thing that hinted to this was it started by saying the Win 95 computer was "active" on a network.--and my answers did not say "could see" but rather "could access resources on"

The Key here is to realize that Win 95 cannot route IPX/SPX across the two cards--so eliminate any answers that say Q or X access the Novell servers.

This leaves two (on my test--not above example) then realize that both had same Novell client this allowed me to eliminate one more answer which treated Q and X differently. Only leaving one possible answer...

That Q and X could access resources on the Y-win 95 machine, and Y could access resources on the Novell server. got it right! 100% on this category.

Answer mentioned was d

Why it is not option "c", since it is possible to modify the Registry, through policy editor, since Registry is nothing but System.dat.

a) the file was written to the disk under the win95 protected-mode file system and has become corrupted
b) the file has long filename that cannot be found under Microsoft MS-DOS
c) the drivespace device driver is not loaded in MS-DOS mode.
d) the file has been fragmented by the VFAT file system so that the Microsoft MS-DOS FAT file system cannot read it.

Answer mentioned was option "c"

My Opinion:
Since Longfile name is not recognized in MS Dos Mode, option "b" is correct.

No, MS Dos Mode will truncate the long file name and see the app as an 8.3 file name: plus no name is given. C is definitely correct--if you can't read the compressed drive then you can't read the drive.


15) Maria's Plug and Play network card is able to use IRQ 5, 7 or 11. Her Plug and Play sound card uses IRQ 3 and her modem is hard-wired to IRQ 7. Which IRQ will be assigned first during the system startup?

a) IRQ 3
b) IRQ 5
c) IRQ 7
d) IRQ 11

My Opinion:
Since Legacy devices has to be handled before Plug and Play cards the option "c" is correct. Plug and Play cards can be automatically switched over to unused IRQ's after the legacy cards are set.

Yep, hard-wired is first!
