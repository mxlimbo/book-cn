《 对《用卷拷贝》一文的补充 》

{　　贵报今年第2期刊登的云南袁光熙先生的《用卷拷贝》(全称为《用卷拷贝的方法移植Novell网服务器》)一文，读后很受启发。原文中提到移植完毕，除超级用户和guest用户及everyone用户组外，其他所有用户及用户组均被取消，所有的文件和目录权限均被取消。事实上，问题出在XCOPY的功能上。因为，所有Windows 9x、DOS下的XCOPY都不支持对隐含文件的复制（Windows NT下的XCOPY支持）。而Novell NetWare 中所有用户及用户组信息，所有的文件和目录权限设置信息等，恰恰存放在服务器硬盘SYSTEM子目录下的三个隐含文件里面，这三个隐含文件分别是：NETSOBJ.SYS、NET SPROP.SYS、NET SVAL.SYS。
　　事实上，利用一台有盘工作站作“缓存盘”比较方便，另外新硬盘DOS分区未必和原硬盘的DOS分区容量完全相同。
　　1利用一台有盘工作站将原硬盘数据（含隐含文件）全部备份（最好为Windows 9X工作站，可以利用资源管理器复制所有文件）；同时，备份Novell启动文件。
　　2更换为新硬盘，在新硬盘上建立一合适容量的DOS分区，装入DOS操作系统，并将备份的Novell启动文件拷入。
　　3运行SERVER.EXE，利用LOAD INSTALL 进行分区、建卷SYS、安装卷。
　　4利用超级用户，登录新系统，将备份的原硬盘数据恢复到新的系统上即可。
　　（山东　李景海）}
