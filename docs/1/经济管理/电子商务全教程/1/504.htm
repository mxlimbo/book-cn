第四页：跟踪每件事

　　为了跟踪顾客，需要为每个旗帜广告赋一个代码，然后把这个代码用在把你的站点链接到广告上的URL中。URL如下所示：

　　http://www.ramforless.com/index.cfm?MC=WM001

　　要看这样做如何跟踪顾客的例子，可以访问http://www.ramforless.com-advertisements。如果你点击，可以发现网站记录了你从哪里来。这是因为MC=WM001 是一个记录你从Webmonkey网页来的媒体代码。媒体代码在顾客的用户连接中被跟踪，如果客户填了一个定单，这个代码就被附加在以后的定单报告中。也可以在电子邮件中嵌入这样的媒体代码。

　　在ramforless.com，关于顾客、定单和广告的所有信息都存储在同一个数据库中，用这个数据库可以更容易地产生报告。

"销售代价" 报告
代码来源 销售 代价 销售/$1 代价 占销售的%
111111 2,176 1,000 2.18 45.96 %
222222 2,046 1,000 2.05 48.88 %
333333 3,160 1,000 3.16 31.65 %
444444 10,690 1,000 10.69 9.35 %
555555 3,150 1,000 3.15 31.75 %
WM001 2,934 1,000 2.93 34.08 %
直接 18,238 1,000 18.24 5.48 %

　　这个报告显示了来自于不同媒体代码（111111、222222、WM001等 ）的销售额。"代价"是指在做旗帜广告、电子邮件上面的花费。"销售/$1代价"指每花费一美圆的销售额。"占销售的%"是指"代价"与"销售"的比值。

　　所以对于来自于广告"11111"的一美圆的销售，45.96美分用在做广告上。如果你的销售代价是45.96%，你的生意将很艰辛。另一方面，广告"44444"的销售代价是岁入的9.35%，你可以从中继续获利。

　　也可以跟踪不用任何特定的广告直接进入网站的顾客。在上面的例子里，媒体代码"直接"显示了口头仍旧是产生低代价销售的最好的资源。

　　但这并不意味着你应该放弃所有的广告和只依赖于口头传播。使用日志文件跟踪用户活动的一个致命的缺陷是它不能对那些通过广告进入你的网站，然后在其它站点购物后回到你的网站购物的活动做出跟踪。跟踪系统将把这种情况看做广告的失败，而作为"直接"的胜利，实际上不是这种情况。当增强网站时，很多事通常不是全部或一无是处。通常，最有效的市场策略涉及到不同方法的正确合成。
