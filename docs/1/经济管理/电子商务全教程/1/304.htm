第四页：如何付费？

　　付费时有多项选择。信用卡？记帐卡？购物定单？客户帐号？但是电子商务的大部分事务处理是基于信用卡的。

　　在这些情况后面，信用卡事务处理相当复杂。涉及到大量独立的部门，包括你（商人）、你的银行、顾客、顾客的银行、发行客户信用卡的公司、还有管理整个局面的大型信用和债务"获得银行"。在顾客的钱进入你的帐户之前所有这些不同的部门需要合作。为了保证这些人合作愉快，还有很多事要考虑。

　　最先考虑的是：别从信用卡中收费 - 直到产品被发送到顾客手里。"喔？"你说道，"但是我在他们给了信用卡号之后就可以从卡中收费了，不是吗？" - 错了。除非你以前从事过邮件定单商业，你就不会熟悉这个规则。在技术上，你不允许立即把钱转移到你的帐户 - 除非产品在当天发出。在实际的商店里，送货和付费同时发生。商人从顾客的卡中收钱，然后顾客的包中装满东西走出商店。但是当你在远程卖东西时，你就好通过互联网（除非你在分发软件或即时在线内容），你不会从信用卡中赊帐 - 除非你已经完成了整个"挑选、打包、运输"执行过程。所以下面是大多数互联网上信用卡交易的过程：

  1. 鉴定。要保证你收到的信用卡有有效的号码，被实际发行，和没有报失。
  2. 授权。这个过程检查购买的基金是否可用。如果可用，你可以预定这些资金。但是等等 - 你还没得到这些钱。
  3. 结算。一旦你已经运出或递送产品给你的顾客，那么就应该让银行知道。银行将释放先前预定的资金，这些资金将穿越若干家银行和中介到达你的帐户。

　　这就是大多数互联网交易的过程。但是有很多障碍。例如，如何处理退货？如何处理部分销售？如何处理返还的定单？如何满足部分定单？当选择维护付费系统的方案时应该考虑这些问题。

  • CyberSource.

　　有这么多钱在互联网上飞来飞去，很多顾客和商人担心交易可能不安全，众所周知的黑客在偷每个人的钱。上面列出的很多方案使用一种新的SET（安全电子交易 - secure electronictransaction）标准，它包含额外的安全措施和防骗技术，在确保你的交易是安全的方面做了很多工作。
