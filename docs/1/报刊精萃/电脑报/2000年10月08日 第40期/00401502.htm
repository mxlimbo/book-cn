《 {修改INI文件更正初始化错误} 》

{　　Kinetix公司发行的3D Studio MAX和3D Studio VIZ系列软件提供了多种显示驱动选项，可在“File”菜单下Preference\Viewports\Choose Driver进行选择，如果选择了该驱动程序而又没有相应设备，在下次启动运行时则会出现提示：程序不能正常显示或不能启动运行。
　　解决办法是重新设置成默认显示驱动程序。回到“File”菜单下Preference\Viewports\Choose Driver，选择系统默认设置HEIEI－Software Z Buffer项，此选项可用在符合最低系统需求的所有显示卡上，重新运行3D Studio MAX／VIZ即可；若3D Studio MAX／VIZ不能启动，因而也就无法进入操作界面进行设置，此时通常的做法是把软件重装一遍，其实大可不必。不能启动是因为程序在初始化时所选项目与实际配置不符，造成初始化失败，而程序初始化配置放在INI文件当中，只要打开应用程序的INI文件，将其值改为相关配置即可（3D Studio MAX／VIZ提供SZB5.HDI和GDI5.HDI两种着色显示驱动程序，其位置是在程序文件的drivers子文件夹中）。分别打开3D Studio MAX／VIZ目录下3dsmax.ini／3dsviz.ini文件，把[windowstate]节中下列三项改为：
　　GFXType=HEIDI
　　GFXRenderer=SZB5.HDI
　　GFXDevice=GDI5.HDI
　　保存退出，然后启动运行软件便万事大吉。
　　把3D Studio MAX／VIZ安装到其它非C盘上时也会遇到程序初始化错误，其原因是3D Studio MAX／VIZ默认安装路径是C盘，其初始化配置文件设置也是C盘，若安装到其它盘上，在启动运行时在C盘上找不到初始化文件，程序不能初始化，当然也就不能启动。只要打开3D Studio MAX／VIZ程序中3dsmax.ini／3dsviz.ini和plugin.ini两个文件，把所有“C:”盘符改为程序安装所在盘符便可启动运行。
　　INI初始化配置文件包含了程序运行环境所需要的重要信息，它包括其自身所定义的以及满足系统硬件需求的一些设定，任何一方面错误都可能造成程序运行不正常或不能启动。通过对初始化文件结构的分析，也就可以方便地解决诸如上面提到的一些问题。（文／杨嘉）}
