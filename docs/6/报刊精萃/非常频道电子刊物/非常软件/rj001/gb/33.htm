□...Sun公布秘密武器——Jxta

摘自：ZDNet China

　　本周四，Sun揭示了一个旨在削弱微软霸权的工程-Jxta工程。Jxta是Sun对对等技术（peer to peer）的贡献。

Jxta工程是对 .Net 的直接威胁。Sun让像 Windows这样的操作系统降低了地位，而让软件在最高的层次上工作。Sun希望能把微软最大的敌人之一-公开源代码社区拉拢进Jxta计划。 Jxta 将是一个公开源代码的软件，这意味着任何人能没有限制的修改并发布该软件。对Sun来说，它是一个对等的基础架构，在此之上商业应用程序能够运行。

Jxta包括了对等网的设备识别身份并组织到一起的标准。它也包括一个保证分布式程序不伤害它们运行于其上的设备的安全机制。计算任务能连接在网络的一个"管道"中。 Jxta还提供了一个任务监视并控制的机制。Jxta可以成为Sun未来的Sun One平台的一部分，但最终它可能变成象(微软的) .Net那样复杂的东西。

Sun的首席科学家Bill Joy否认有任何控制新市场的企图。他说 Jxta 只是提供一个公共语言让各种对等网应用程序能在一起协作。"我们不想要有一个标准组织，"Joy说，"我们不要让每个人获得它的许可，就像我们对 Java做的那样。我们甚至不想成为它的中心。"

Jxta的第一个版本将首先发布在4月份 CollabNet站点上。

目前，Jxta 已经有了竞争对手。Intel 已经向一个对等技术应用标准的工作组提供了支持。尽管这一举动受到了争议，这个工作组已经开始了工作。

毫无疑问Sun希望Jxta比它的前辈Java和Jini更成功。Sun一直希望能吸引公开源代码的社区。但是要利用公开源代码的程序员的能量并不容易。

1994年发布的Java是Sun抛开微软的第一次尝试。 Java 的承诺是让程序运行在任何类型的计算机上，这一个诺言在7年以后也只是完成了一部分。 Jini在 1999年发布，其目的是让数码相机和打印机等设备在没有计算机充当中间人的情况下也能通讯。 Jini 基本也是一个商业失败。Joy说 Jxta能在支持Java的设备上运行得很好，但是它不要求 Java作为基础。

在对等世界里，已经有了一些成功的范例 。但是在许多雄心勃勃的开发者的脑子里，今天的因特网应该发展到一个新的阶段。他们希望有一个更复杂的网络，其中单个的计算机，无线电话，强大的服务器和数据库在一起工作提供新的网络服务，不管是远程的软件服务或象 Napster 和 Gnutella 一样的交互的文件共享程序。
