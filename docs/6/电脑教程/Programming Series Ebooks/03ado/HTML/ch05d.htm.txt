<A NAME="177"><H1>ADO Parameters Collection</H1></A>

<p>Each Command object exposes a Parameters collection that you can use
for parameterized queries. The Parameters collection exposes most of
the same properties and methods as all standard collections, but there
are two methods on the Parameters collection worth
discussing&#8212;<i>Append</i> and *<i>Refresh</i>. Each of these
methods can be used to populate the Parameters collection. Generally,
you'll use either one or the other in your applications.</p>

<table cellpadding="5" border="0" width="95%">
<tr><th colspan="2">Important Methods of the Parameters Collection</th></tr>
<tr><th><i>Method Name</i></th><th><i>Description</i></th></tr>
<tr><td valign="top"><i>Append</i></td><td valign="top">Appends a Parameter object to the Parameters collection</td></tr>
<tr><td valign="top"><i>Refresh</i></td><td valign="top">Refreshes the Parameters collection</td></tr>
</table>

<A NAME="178"><h2><i>Append</i> Method</h2></A>
<p>Use the <i>Append</i> method to add a Parameter object to the
Parameters collection. By populating the Parameters collection on your
own, you can achieve better performance than if you have ADO ask the
OLE DB provider to supply that information. <i>Append</i> takes one
parameter:</p>

<ul>
<p><li> <i><b>Object</b> </i>This value specifies the
Parameter object that you want to append to the Parameters
collection.</li></p>
</ul>
