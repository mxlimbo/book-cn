<A NAME="138"><H1>Error Checking and Handling</H1></A>

<p>We'll first cover error checking and error handling, as they are vital to writing a successful Winsock application. It is actually common for Winsock functions to return an error; however, many times the error is not critical and communication can still take place on that socket. The most common return value for an unsuccessful Winsock call is <i>SOCKET_ERROR</i>, although this is certainly not always the case. When covering each API call in detail, we'll point out the return value corresponding to an error. The constant <i>SOCKET_ERROR</i> actually is -1. If you make a call to a Winsock function and an error condition occurs, you can use the function <i>WSAGetLastError</i> to obtain a code that indicates specifically what happened.
