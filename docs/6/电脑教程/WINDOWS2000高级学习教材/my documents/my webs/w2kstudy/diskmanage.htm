磁盘管理概述

磁盘管理程序是用于管理它们所包含的硬磁盘和卷，或者分区的系统实用程序。通过磁盘管理，您可使用文件系统创建卷、格式化卷、初始化磁盘以及创建容错磁盘系统。“磁盘管理”取代了 Windows NT 4.0 及更早版本中使用的“磁盘管理器”实用程序，它提供了许多新的功能，包括：

动态磁盘。使用动态磁盘，您可在不关闭系统或中断用户操作情况下完成管理任务。例如，您可在不重新启动系统的情况下创建卷、扩展卷或制作卷的镜像。也可在不重新启动的情况下添加新的磁盘。多数配置的更改几乎马上就能生效。

本地和网络驱动器管理。可从您网络上的任何 Windows 2000 计算机上管理任何其他网络上运行 Windows 2000 或 Windows NT 4.0 并且您是其管理员的计算机。

简化的任务和直观的用户界面。“磁盘管理”程序易于使用。右键单击菜单为您显示能对所选对象执行的任务，而向导则指导您创建分区或卷以及初始化或升级磁盘。

装入的驱动器。可使用“磁盘管理”在本地 NTFS 卷上的任何空文件夹中连接或装入本地驱动器。装入驱动器使数据更易访问并根据您的网络环境和系统使用情况为您提供管理数据存储的灵活性。

 

卷和分区

在动态磁盘上，存储区被分成卷而不是分区。您可在任何时候从基本存储器升级到动态存储器。升级至动态存储器时，现有的分区将转换成卷

 

将基本磁盘升级为动态磁盘

将要升级的任何磁盘在磁盘末尾都必须至少包含 1 MB 的可用空间，以使升级成功。在磁盘上创建分区或卷时，“磁盘管理”程序自动保留这一可用空间，但带有其他操作系统创建的分区或卷的磁盘可能没有这样的可用空间。

在升级磁盘之前，请关闭在这些磁盘上运行的任何程序。

当您将基本磁盘升级到动态磁盘之后，基本磁盘上的任何现有分区都将变为（动态）简单卷。不能将动态卷改回分区。任何现有的镜像卷（镜像集）、带区卷（带区集）、RAID-5 卷（带有奇偶校验的带区集）或跨区卷（卷集）分别变成动态镜像卷、带区卷、RAID-5 卷或跨区卷。

升级后，动态磁盘不能包含分区或逻辑驱动器，也不能由 MS-DOS 和除 Windows 2000 以外的其他 Windows 操作系统访问。

可移动媒体。不能将可移动媒休设备升级为动态卷。可移动媒体设备仅包含主分区。

跨越多磁盘的卷。如果基本磁盘包含驻留在多个磁盘中的任何卷（如镜像卷、带区卷、跨区卷或 RAID-5 卷），则您也必须升级包含这些卷的所有其他磁盘。如果其他的这类磁盘包含驻留在多个磁盘中的其他卷，则您还必须升级包含这些卷的某些部分的所有磁盘。您必须一起升级所有这些磁盘。这些磁盘的每一个在磁盘结尾处都必须至少有 1 MB 的未分配空间，否则升级将会失败。（即使是在"磁盘管理"程序中见不到，该空间也会存在。）

扇区大小超过 512 字节的磁盘。若磁盘的扇区大小超过 512 字节，则不能将基本磁盘升级为动态磁盘。

重新安装 Windows 2000只有在该卷从基本卷升级后，您才能在动态卷上进行 Windows 2000 的全新安装。（将基本卷升级为动态卷会保留分区表信息。该信息要求在动态卷上执行 Windows 2000 的全新安装。）

引导和系统分区。您可将包含有系统或活动分区的基本磁盘升级为动态磁盘。当磁盘升级后，这些分区变为简单系统或活动卷（在重新启动计算机后）。不能将现有的动态卷标记为活动的。可以将包含引导分区的基本磁盘（包含 Windows 2000 操作系统）升级为动态磁盘。升级磁盘后，引导分区变为简单引导卷（重新启动计算机后）。若该磁盘也包含跨区卷（卷集）、带区卷（带区集）、镜像卷（镜像集）或 RAID-5 卷（带有奇偶校验的带区集），则您不能升级包含系统或引导分区的磁盘。

升级失败。如果从启动盘升级，或者要升级的磁盘中的卷或分区正在使用，则必须重启动计算机才能成功升级。如果出现下列出现下列任一情况，计算机重启动后升级都会失败：

计算机重启动时断开了所有现存的动态磁盘。

计算机重启动时更换了磁盘或磁盘组。（磁盘管理器将检测到磁盘更换，升级随之失败。）

改变了要升级的磁盘的磁盘布局。

在升级时出现磁盘 I/O 错误。

镜像引导和系统卷。在将包含引导和系统分区的磁盘升级为动态磁盘后，您便可将引导和系统卷镜像到另一动态磁盘上。然后，如果包含引导和系统卷的磁盘失败，则您可从包含这些卷的镜像的磁盘中启动计算机。详细信息，请参阅解决引导失败问题。

进行升级时。在计算机重新启动后，系统和引导分区升级为动态卷。所有其他分区和基本卷立即进行升级。但是，如果要升级的磁盘上的分区或基本卷正在使用，则产生一个称作"强制卸下"的事件，它意味着使用该卷的任何程序也将被自动断开。若该卷不能强制卸下（例如，在卷上有一个活动的页面文件），则磁盘的升级只有在您重新启动计算机之后才能完成。

其他操作系统。如果待升级磁盘中的分区包含非当前运行的 Windows 2000 的其他操作系统，那么在升级后您将不能从该操作系统启动计算机。

还原为基本磁盘。在将基本磁盘升级为动态磁盘后，不能将动态卷改回到分区。而必须删除磁盘上的所有动态卷，然后使用"还原为基本磁盘"命令。

 

使用镜像卷

镜像卷是容错卷，它将您的数据复制到两个物理磁盘中。它通过使用卷的副本（镜像）复制卷中所包含的信息来提供数据冗余。镜像位于不同的磁盘上。如果其中一个物理磁盘出现故障，则该磁盘上的数据将无法使用，但系统继续使用未受影响的磁盘进行操作。镜像卷在读操作方面比 RAID-5 卷慢，但在写操作方面比它快。

在 Windows NT 4.0 及其更早的版本中，镜像卷也称作镜像集。

通过在另一磁盘上保留完全冗余的副本，镜像卷可保护磁盘上的卷免受介质故障或可能出现的控制器问题的影响。当镜像卷出现故障时，您必须中断镜像卷，以便将其余的卷作为自己带有驱动器号的独立卷。随后您便可利用另一磁盘上大小相同或更大的未使用空间来创建新的镜像卷。

要创建镜像卷，请使用另一磁盘上的可用空间创建卷。如果第二个卷更大，则其余的空间变为可用空间。相同的驱动器号用于两个卷。现有的任何卷（甚至是系统卷和引导卷）都可以使用相同的或不同的控制器镜像到其他磁盘上大小相同或更大的另一个卷。创建镜像的卷时，最好使用大小、型号和制造厂家都相同的磁盘。

与 RAID-5 卷相比，镜像卷具有更好的整体读写性能。镜像卷与 RAID-5 卷相比还具有另一个优点，即当许多镜像卷出现问题时性能不会损失。由于磁盘空间的使用较少，因此根据每 MB 所花费的美元来看镜像卷更贵一些。但是，由于镜像卷只需要两个磁盘，而 RAID-5 卷需要三个或更多的磁盘，因此镜像卷的进口成本更低。

通过提供重复的数据集（在写入磁盘时会成倍地增加所需要的磁盘数和输入/输出操作），镜像卷减少了发生不可恢复错误的机会。但是，由于两个卷之间的请求要进行 I/O 负载平衡，因此读数据的性能会有所增强。

当您想将镜像卷中的空间用于其他用途时，必须首先中断镜像卷的关系，然后删除其中一个卷。中断镜像卷不会删除这些信息，但首先进行备份更为安全。

如果镜像卷中的卷发生了不能恢复的错误，则您需要中断镜像卷的关系，以便将剩余的卷作为独立卷。随后您可在另一磁盘上重新分配一些可用空间，以创建新的镜像卷。

 

使用 RAID-5 卷

RAID-5 卷是一种容错卷，并且数据和奇偶校验值在三个或更多的物理磁盘上成间歇的带区分布。如果物理磁盘的某一部分出现故障，则您可在其余的数据和奇偶校验值基础上重新创建出现故障的那部分数据。在大多数活动由读数据组成的计算机环境中，RAID-5 卷是解决数据冗余的一个很好的方案。

在 Windows NT 4.0 及更早版本中，RAID-5 卷称为带有奇偶校验的带区集。

“磁盘管理”程序在基本磁盘上提供了 RAID-5 卷的有限支持。您可以修复、重新生成 RAID-5 卷的奇偶校验值和删除 RAID-5 卷，但不能在基本磁盘上创建新的 RAID-5 卷。只能在动态磁盘上创建 RAID-5 卷。

与镜像卷相比，RAID-5 卷具有更好的读性能。但是，当某一部分丢失（例如磁盘发生故障）时，读性能会因为需要用奇偶校验信息来恢复数据而降低。

无论如何，对于需要冗余和主要用于读操作的程序，建议该策略要优先于镜像卷。写性能由于奇偶校验计算而降低。同时，在正常操作期间，写操作要求的内存是读操作的三倍以上。并且，当卷出现故障时，与发生故障之前相比，读操作至少要求三倍以上的内存。两种情况都是由奇偶校验计算引起的。

RAID-5 卷的每个带区都包含一个奇偶校验块。因此，您必须至少使用三个而不是两个磁盘来处理奇偶校验信息。如下图所示，奇偶校验带区在所有的卷上分配以平衡 I/O 负载。

当您想恢复 RAID-5 卷中的空间用于其他用途时，如果您想要重新使用该信息，请确认首先备份了其中的信息，然后删除该卷。

 

磁盘配额概述

卷的磁盘配额跟踪以及控制磁盘空间的使用。系统管理员可将 Windows 配置为：

用户超过所指定的磁盘空间限额时，阻止进一步使用磁盘空间和记录事件。

当用户超过指定的磁盘空间警告级别时记录事件。

启用磁盘配额时，可以设置两个值：磁盘配额限度和磁盘配额警告级别。该限制指定了允许用户使用的磁盘空间容量。警告级别指定了用户接近其配额限度的值。例如，可以把用户的磁盘配额限度设为 50 MB，并把磁盘配额警告级别设为 45 MB。这种情况下，用户可在卷上存储不超过 50 MB 的文件。如果用户在卷上存储的文件超过 45 MB，则把磁盘配额系统记录为系统事件。

可以指定用户能超过其配额限度。如果不想拒绝用户访问卷但想跟踪每个用户的磁盘空间使用情况，启用配额但不限制磁盘空间使用将非常有用。也可指定不管用户超过配额警告级别还是超过配额限度时是否要记录事件。

启用卷的磁盘配额时，系统从那个值起自动跟踪新用户卷使用。但是，磁盘配额不应用到现有的卷用户上。可以通过在“配额项目”窗口中添加新的配额项目将磁盘空间配额应用到现有的卷用户上。

可以在本地卷和网络卷上启用配额，但是只能在从卷的根目录共享以及用 NTFS 文件系统格式化的卷上启用配额。

注意

要支持磁盘配额，磁盘卷必须用 Windows 2000 中的 NTFS 版本格式化。用 Windows NT 4.0 中使用的 NTFS 版本格式化的卷将被 Windows 2000 安装程序自动升级。

要管理卷上的配额，您必须是驱动器所在计算机上的 Administrators 组的成员。

如果卷不是用 NTFS 格式化的或者您不是本地计算机上的 Administrators 组的成员，那么卷的属性页上将不显示“配额”选项卡。

文件压缩不影响配额统计。例如，如果用户 A 限制使用 3MB 的磁盘空间，那么只能存储 3MB 的文件，即使文件是压缩的。
