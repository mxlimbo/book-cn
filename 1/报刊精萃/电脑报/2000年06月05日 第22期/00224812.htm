《 DW3的HTML代码格式化和净化功能 》

　我们在利用Dreamweaver 3（简称DW3）进行网页制作前，可以进行“格式化HTML代码”参数设置，如控制缩进、设置行间隔、设置标记和属性名称的大小写等，从而在创作文档时，保持其源代码格式简洁明了，也便于自我维护。
　　利用“格式化HTML代码”和“净化HTML代码”的特性，还可以对现有文档的HTML代码进行格式化和净化，实现统一的HTML代码格式，删除无用的代码，修复代码错误，优化代码，提高代码质量。
　　1.HTML格式化参数设置
　　打开“Edit”菜单，选择“Prefernces”，打开参数选择对话框，选择“HTML Format”项，设置相关参数。如图1。
　　Indent：缩进方式。
　　Use：选择用空格或制表符（Tab）实现缩进。
　　Table Rows and Columns：选择，则会自动缩进用于创建表格的tr标记和td标记。
　　Frames and Framesets：选择，则会自动缩进用于创建框架和框架集的frames标记和framesets标记。
　　Indent Size：单位“字符”。
　　Tab Size：单位“制表符”。
　　Automatic Wrapping：选择自动换行和行数。
　　Line Breaks：不同的操作系统，选择不同的方式。
　　Case for Tage：选择大小写方式，只对文档窗口编辑操作有效。
　　Case for：选择大小写方式，只对文档窗口操作有效。
　　Override Case Of：全选中，则会根据上面的设置，在各种编辑状态下，强制对各种文档的标记或属性的大小写进行重写。
　　Centering：居中方式的两种源代码的选择。
　　2.格式化现有文档的HTML代码
　　通过格式化参数设置，除了“Override Case Of”属性以外，通常只对新建文档或现有文档中新输入的内容有效，对现有文档中的源代码进行格式化，操作如下：
　　用DW3打开现有文档，打开“Commands”菜单，选择“Apply Source Formatting”命令。
　　3.HTML重写参数设置
　　打开“Edit”菜单，选择“Prefernces”，打开参数选择对话框，选择“HTML Rewriting”项，设置相关参数。如图2。
　　Fix Invalidly Nested and Unclosed Tags：修复无效的嵌套和未封闭标记。
　　Remove Extra Closing Tags：删除多余的封闭标记。
　　Warn when Fixing or Removing Tags：当修复或删除标记时发出警告。
　　Never Rewrite HTML :In Files with Extensions：从不重写带有以下扩展名的HTML文档，可以在文本框中输入相应的扩展名，用空格隔开。
　　Special：两个选项用于设置是否对某些特定字符进行编码。
　　4.净化现有文档的HTML代码
　　净化现有文档的HTML代码，可以删除无用的代码，优化源代码。如下操作：
　　用DW3打开现有文档，打开“Commands”菜单，选择“Clean Up HTML”命令。出现如图3对话框，进行选择。
　　Empty Tags：空标记。
　　Redundant Nested Tags：多余嵌套标记。
　　Non－Dreamweaver HTML Comments：所有非Dreamweaver自动生成的注释信息。
　　Dreamweaver HTML Comments：Dreamweaver HTML注释。
　　Specific Tag(s)：可以在文本框中输入想删除的标记。
　　Combine Nested<font> Tags When Possible：对文档中的嵌套<font>标记进行重新组合。
　　Show Log on Completion：完成后显示提示。
　　另外，DW3还可以对Microsoft Word生成的HTML文档进行净化，不妨自己试试。

　　
（上海　赵晓旦）
